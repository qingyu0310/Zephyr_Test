# 波特率问题分析

## 现象

Overlay 中配置 `bus-speed = <1000000>` 或 `bitrate = <1000000>`，实际生效的波特率始终是 125000。

## 根因

HPMicro Zephyr glue 层驱动 `mcan_hpmicro.c` 的 `HPM_MCAN_INIT(n)` 宏定义有一处 copy-paste bug。

```c
#define HPM_MCAN_INIT(n)                     \
    ...
    static const struct hpm_mcan_config hpm_mcan_cfg_##n = { \
        .common = CAN_DT_DRIVER_CONFIG_INST_GET(inst, 0, 1000000), \  // ← BUG
        .base = (MCAN_Type*)DT_INST_REG_ADDR(n), \
        ...
    };
```

宏的参数是 `n`（表示 DT 实例编号），但第 1038 行错误地用了字面量 `inst` 而非参数 `n`。

展开过程：

```
CAN_DT_DRIVER_CONFIG_INST_GET(inst, 0, 1000000)
→ CAN_DT_DRIVER_CONFIG_GET(DT_DRV_INST(inst), 0, 1000000)
→ CAN_DT_DRIVER_CONFIG_GET(DT_N_INST_inst_hpmicro_hpm_mcan, 0, 1000000)
```

`DT_N_INST_inst_hpmicro_hpm_mcan` 这个节点**不存在**，所有 DT 属性读取都失败，回退到默认值：

```c
.bitrate = DT_PROP_OR(node_id, bitrate,
    DT_PROP_OR(node_id, bus_speed, CONFIG_CAN_DEFAULT_BITRATE)),
// 节点不存在 → 读取失败 → 使用 CONFIG_CAN_DEFAULT_BITRATE = 125000
```

## 修复

`CAN_DT_DRIVER_CONFIG_INST_GET(inst, ...)` → `CAN_DT_DRIVER_CONFIG_INST_GET(n, ...)`

```c
.common = CAN_DT_DRIVER_CONFIG_INST_GET(n, 0, 1000000),  // 已修正
```

同个结构体初始化的其他 DT 宏调用都正确用了 `(n)`，唯独这一行写错成了 `inst`。

## 验证

修复后输出：

```
CAN: clock=80000000 baud=1000000 NBTP=0x26003a13
```

- NBRP=0 → prescaler=1
- TSEG1=59, TSEG2=20
- total TQ = 1 + 59 + 20 = 80
- bitrate = 80MHz / 1 / 80 = 1Mbps
- sample point = 75%
