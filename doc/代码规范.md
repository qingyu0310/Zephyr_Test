# 代码规范

---

## 一、命名规范

### 1.1 命名风格总表

| 类别 | 风格 | 示例 | 说明 |
|------|------|------|------|
| 类/结构体 | `PascalCase` | `DjiC6xx`, `PowerCtrl`, `WheelModule` | 首字母大写 |
| 枚举类 | `PascalCase` | `enum class DFirst` | |
| 枚举值 | `PascalCase` | `Disable`, `Enable` | |
| 命名空间 | `snake_case` | `thread::chassis`, `alg::pid` | |
| 文件作用域常量 | `kPascalCase` | `kTorqueK`, `kTotalBudget`, `kPeriodMs` | 前缀 `k` |
| 文件作用域全局变量 | `g_snake_case` | `g_vx`, `g_wh_target`, `g_k_factor` | 前缀 `g_` |
| 公开 API 函数 | `snake_case` | `thread_init`, `thread_start`, `SetTarget` | |
| 文件内静态函数 | `PascalCase` | `SwerveKinCalculate`, `ReadRemote` | |
| 类成员函数 | `PascalCase` | `ReadAll`, `GetPower`, `CalcAngle` | |
| 宏 | `UPPER_SNAKE_CASE` | `USE_POWERMETER`, `N_Wheel` | |
| 结构体成员 | `snake_case` | `cfg.k1Init`, `snap.angle`, `msg.tx_id` | |

### 1.2 常量前缀约定

```
kPrefix     → 文件/函数作用域 constexpr（k + PascalCase）
              kTorqueK, kTotalBudget, kPeriodMs
g_prefix     → 文件作用域全局变量（g_ + snake_case）
              g_vx, g_wh_target, g_k_factor
```

> 注意：部分旧常量使用 `K` 大写前缀（如 `KMaxMoveVelocity`），新代码统一使用小写 `k` 前缀。

### 1.3 命名空间约定

```
thread::模块名       → 线程层（如 thread::chassis, thread::can）
instance::模块名     → 实例化数据（如 instance::chassis）
alg::模块名          → 算法层（如 alg::pid, alg::power_ctrl）
topic::模块名        → zbus 消息（如 topic::chassis_to_can）
```

### 1.4 文件名前缀

```
trd_       → 线程文件（thread 的缩写）
              trd_chassis.cpp, trd_can_tx.cpp, trd_gpio.cpp
dev_       → 驱动/设备文件（已不再使用，新驱动直接命名）
```

## 二、文件结构

### 2.1 线程文件固定模式

每个线程提供两个 API：

```cpp
namespace thread::模块名 {

static Thread<> thread_{};                      // 线程实例（文件作用域 static）

void thread_init()
{
    // ① 驱动初始化
    // ② PID 参数配置
    // ③ 外设初始化
}

void thread_start(uint8_t prio, void* p2, void* p3)
{
    thread_.Start(Task, prio, ...);
}

} // namespace thread::模块名
```

在 `System_startup.cpp` 中依次调用：

```cpp
void System_Modules_Init()   { 模块A::thread_init(); 模块B::thread_init(); }
void System_Thread_Start()   { 模块A::thread_start(); 模块B::thread_start(); }
```

### 2.2 头文件模板

```cpp
/**
 * @file trd_xxx.hpp
 * @author qingyu
 * @brief 一行说明
 * @version 0.1
 * @date YYYY-MM-DD
 *
 * @copyright Copyright (c) YYYY
 *
 */

#pragma once

#include <cstdint>

// 常量 / 配置
constexpr uint8_t N_Wheel = 2;

// 结构体 / 类定义
struct WheelModule {
    DjiC6xx steer_motor{};
};

// 实例化（inline 避免 ODR 冲突）
inline WheelModule chassis_wheel[N_Wheel] {};

// API 声明
namespace thread::xxx {
    void thread_init();
    void thread_start(uint8_t prio = 5, ...);
};
```

- 使用 `#pragma once` 而非 `#ifndef` 传统宏
- 实例化数据放在 `instance::模块名` 命名空间，用 `inline` 修饰

### 2.3 源文件结构

```
① include 头文件     → 自身 hpp 优先
② include 依赖       → 按逻辑分组，空行分隔
③ 匿名 namespace     → 文件内私有常量和函数
④ 具名 namespace     → 模块实现
```

```cpp
#include "trd_xxx.hpp"
#include "依赖1.hpp"
#include "依赖2.hpp"

namespace {

constexpr float kPi = 3.1415926535f;
float HelperFunction(float a) { ... }

} // namespace

namespace thread::xxx {

using namespace instance::xxx;

static Thread<> thread_{};

// 常量
static constexpr float kSomeConstant = 1.0f;

// 全局变量
static float g_some_value = 0.0f;

// 静态函数
static void StageOne() { ... }

// Task 函数
static void Task(void*, void*, void*) { ... }

// API
void thread_init() { ... }
void thread_start(...) { ... }

} // namespace thread::xxx
```

## 三、代码风格

### 3.1 缩进与空格

- 缩进：**4 空格**（不使用 Tab）
- 行尾：**Unix LF**
- 行尾不留空格

### 3.2 括号风格

函数定义左大括号**换行**，控制语句左大括号**不换行**：

```cpp
// 函数定义：换行
static void SwerveKinCalculate()
{
    // ...
}

// 控制语句：不换行
for (uint8_t wi = 0; wi < N_Wheel; wi++) {
    const float vx_w = ...;
    if (spd > 1e-6f) {
        g_wh_target[wi].angle = ...;
    }
}
```

### 3.3 作用域内变量声明

```cpp
// 变量在首次使用时声明，就近原则
const auto snap = chassis_wheel[wi].steer_motor.ReadAll();
const float chassis_angle = NormalizeAngle(-snap.angle);
```

### 3.4 初始化

```cpp
// 聚合初始化用 {}
DjiC6xx::Config cfg{};
cfg.rx_id = 0x202;

// 零初始化用 {}
float g_k_factor[N_Wheel] {};

// 简单的值初始化用 =
float pi = 3.14f;
```

### 3.5 命名空间别名

```cpp
// 在 cpp 文件的 namespace 内部使用 using，不在头文件使用
namespace thread::chassis {
using namespace instance::chassis;
```

## 四、注释规范

### 4.1 Doxygen 注释

公开接口使用 `/** @brief ... */` 格式，可选 `@param`、`@return`：

```cpp
/**
 * @brief 优劣弧判断：对比当前角度与目标角度，必要时翻转目标并反转行进方向
 * @param current 当前底盘角度 (rad)
 * @param target 目标角度 (rad)，若走劣弧则修改为翻转后的目标
 * @return 方向因子：1.0 正转 / -1.0 反转
 */
static float OptimalArc(float current, float& target);
```

### 4.2 单行注释

```cpp
// 实现逻辑注释（不解释"是什么"，解释"为什么"）
// 走短路径：目标角度 ±π，行进电机反转
target = NormalizeAngle(current + err - (err > 0.f ? kPi : -kPi));
```

### 4.3 尾部注释

对齐到同一列，解释变量用途：

```cpp
static constexpr float kTorqueK = 0.3f;     // C6xx 转矩常数 N·m/A
static float g_vx = 0.0f;                   // 底盘速度指令 X
static float g_vy = 0.0f;                   // 底盘速度指令 Y
```

### 4.4 域分隔注释

长文件中用空行 + 注释分隔逻辑段落：

```cpp
/**
 * @brief 功率预测 + 功率分配（转向组优先，行进组分剩余）
 */
inline static void PowerAlloc()
{
```

## 五、常量与配置

### 5.1 constexpr 优先

```cpp
static constexpr float kTorqueK = 0.3f;     // ✓ 正确：编译期常量
const float kTorqueK = 0.3f;                // × 避免：运行时 const 不如 constexpr 严格
#define TORQUE_K  0.3f                       // × 避免：除非是条件编译
```

### 5.2 宏仅用于条件编译

```cpp
#define USE_POWERMETER      1     // ✓ 正确：条件编译开关
#if USE_POWERMETER
    SteerPwrMeter.Init(KSteerPwrMeterId);
#endif
```

## 六、线程安全

### 6.1 中断上下文 → 控制线程

使用 **seqlock**（`atomic_t seq_`）保护多字段一致快照：

```cpp
// 写入（中断上下文）
void CanCpltRxCallback(uint8_t* rx_data) {
    atomic_add(&seq_, 1);  // 开始：seq 变为奇数
    now_angle_ = ...;
    now_omega_ = ...;
    atomic_add(&seq_, 1);  // 结束：seq 变为偶数且递增
}

// 读取（控制线程）
Snapshot ReadAll() const {
    Snapshot snap;
    do {
        seq = atomic_get(&seq_);
        if (seq & 1) continue;         // 写进行中，重试
        snap.angle = now_angle_;       // 批量读取
        // ...
    } while (atomic_get(&seq_) != seq); // 检查是否被写打断
    return snap;
}
```

### 6.2 单字段原子访问

对于 32-bit RISC-V 平台，aligned float 读写是天然的原子操作，单字段访问不需要锁：

```cpp
float GetNowAngle() const { return now_angle_; }  // ✓ 正确：单字段原子
```

### 6.3 线程间通信

层间通过 **zbus** 发布订阅传递数据，不共享全局变量：

```
中断层 → zbus_pub → 控制层（订阅者）
控制层 → zbus_pub → 发送层（订阅者）
```

同一线程内各阶段之间通过全局变量（`g_` 前缀）传递中间结果。

## 七、Kconfig 体系

### 7.1 前缀约定

| 前缀 | 域 | 示例 |
|------|----|------|
| `TRD_` | 线程 | `TRD_CHASSIS` |
| `CTL_` | 控制器 | `CTL_PID`, `CTL_POWER` |
| `DEV_` | 设备驱动 | `DEV_MOTOR_DJI` |
| `TPC_` | 消息通道 | `TPC_MOTOR_CMD` |
| `FLT_` | 滤波器 | `FLT_LPF` |
| `ID_` | 辨识算法 | `ID_RLS` |
| `COM_` | 通信 | `COM_CAN` |
| `SHELL_` | 命令行 | `SHELL_PID` |

### 7.2 依赖管理

在 `Kconfig` 中用 `select` 表达依赖，每个线程 `select` 其所需组件：

```
config TRD_CHASSIS
    bool "Chassis control thread"
    select CTL_PID
    select CTL_POWER
    select DEV_MOTOR_DJI
    select SHELL_PID
```

## 八、目录规范

```
projects/thread/       → 线程实现（trd_xxx.cpp/h）
topic/                 → zbus 消息定义
algorithm/             → 算法（pid / power_ctrl / rls / lpf / timer）
drivers/               → 硬件驱动（can / gpio）

cmd/shell/             → shell 调试命令
doc/                   → 文档
```

## 九、常见反模式

| 反模式 | 正确的做法 |
|--------|-----------|
| 头文件中 `using namespace` | cpp 的具名 namespace 内部使用 |
| 魔法数字硬编码 | `constexpr` 命名常量 |
| `#define` 代替常量 | `constexpr` 变量 |
| 全局变量跨线程共享 | zbus 发布订阅，或 `g_` 前缀局部于单线程文件内 |
| 阻塞等待输入源 | `K_NO_WAIT` + 保持上次有效值 |
| 固定值 `k_msleep` 控制周期 | 测量耗时 + 动态补偿 |
