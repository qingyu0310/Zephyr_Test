# 代码规范

---

## 一、命名规范

### 1.1 命名风格总表

| 类别 | 风格 | 示例 |
|------|------|------|
| 类 / 结构体 | `PascalCase` | `DjiC6xx`, `PowerCtrl`, `WheelModule` |
| 枚举类 | `PascalCase` | `enum class DFirst` |
| 枚举值 | `PascalCase` | `Disable`, `Enable` |
| 命名空间 | `snake_case` | `thread::chassis`, `alg::pid` |
| 文件作用域常量 | `kPascalCase` | `kTorqueK`, `kTotalBudget` |
| 文件作用域全局变量 | `g_snake_case` | `g_vx`, `g_wh_target` |
| 公开 API | `snake_case` | `thread_init`, `thread_start` |
| 文件内静态函数 | `PascalCase` | `SwerveKinCalculate`, `ReadRemote` |
| 类成员函数 | `PascalCase` | `ReadAll`, `GetPower`, `CalcAngle` |
| 宏 | `UPPER_SNAKE_CASE` | `USE_POWERMETER` |
| 结构体成员 | `snake_case` | `cfg.k1Init`, `snap.angle` |

### 1.2 常量前缀

```
kPascalCase    → 文件/函数作用域 constexpr     kTorqueK, kTotalBudget
g_snake_case   → 文件作用域全局变量               g_vx, g_wh_target
```

旧代码中有 `KMaxMoveVelocity` 等大写 K 前缀，新代码统一用小写 `k`。

### 1.3 命名空间约定

```
thread::xxx     → 线程层（thread::chassis, thread::can）
instance::xxx   → 实例化数据（instance::chassis）
alg::xxx        → 算法层（alg::pid, alg::power_ctrl）
topic::xxx      → zbus 消息（topic::chassis_to_can）
```

### 1.4 文件名前缀

```
trd_   → 线程文件   trd_chassis.cpp, trd_can_tx.cpp
算法文件直接命名     pid.cpp, rls.hpp
驱动文件直接命名     output.hpp, uart.hpp
```

## 三、文件结构

### 3.1 线程文件固定模式

```cpp
// trd_xxx.hpp
#pragma once

namespace thread::xxx {
    void thread_init();
    void thread_start(uint8_t prio = 5);
}
```

```cpp
// trd_xxx.cpp
#include "trd_xxx.hpp"

namespace {
// 文件内私有常量、辅助函数
constexpr float kPi = 3.1415926535f;
} // namespace

namespace thread::xxx {

static Thread<> thread_{};              // 线程实例（文件作用域 static）
static /* ... */ g_xxx = {};            // 文件内全局变量

static void StageOne() { /* ... */ }    // 阶段性函数
static void Task(void*, void*, void*)   // 主循环
{
    for (;;) {
        StageOne();
        k_msleep(1);
    }
}

void thread_init() { /* 初始化 */ }
void thread_start(uint8_t prio) { thread_.Start(Task, prio); }

} // namespace thread::xxx
```

### 3.2 头文件模板

```cpp
/**
 * @file trd_xxx.hpp
 * @author qingyu
 * @brief 一行说明
 */

#pragma once

#include <cstdint>

// 常量
inline constexpr uint8_t kFoo = 2;

// 结构体
struct WheelModule {
    DjiC6xx steer_motor{};
};

// 实例化（inline 避免 ODR 冲突）
inline WheelModule chassis_wheel[2] {};

// API 声明
namespace thread::xxx {
    void thread_init();
    void thread_start(uint8_t prio = 5);
}
```

### 3.3 include 顺序

```cpp
#include "自身头文件优先"     // 确保 hpp 独立可编译
#include <zephyr/zbus/zbus.h>  // 标准库 / Zephyr 头文件
#include "project/依赖.hpp"    // 项目内部依赖
```

## 四、代码风格

### 4.1 基础规则

- 缩进：**4 空格**（不使用 Tab）
- 行尾：**Unix LF**，行尾不留空格
- 函数定义左大括号**换行**，控制语句左大括号**不换行**

```cpp
// 函数
static void SwerveKinCalculate()
{
    for (uint8_t i = 0; i < N_Wheel; i++) {
        if (spd > 1e-6f) {
            g_target[i] = atan2f(vx, vy);
        }
    }
}
```

### 4.2 变量声明

```cpp
// 就近声明，首次使用时定义
const auto snap = motor.ReadAll();
const float angle = NormalizeAngle(-snap.angle);
```

### 4.3 初始化

```cpp
// 聚合/零初始化用 {}
DjiC6xx::Config cfg{};
float arr[N] {};

// 简单值用 =
float pi = 3.14f;
```

## 五、常量与配置

### 5.1 constexpr 优先

```cpp
static constexpr float kTorqueK = 0.3f;   // ✓ 编译期常量
#define TORQUE_K 0.3f                      // ✗ 仅用于条件编译
```

### 5.2 namespace 作用域常量

```cpp
inline constexpr uint8_t kMaxMotors = 4;   // ✓ C++17 外部链接
static constexpr uint8_t kMaxMotors = 4;   // ✗ 每个 TU 独立拷贝
```

### 5.3 宏仅用于条件编译

```cpp
#define USE_POWERMETER  1
#if USE_POWERMETER
    SteerPwrMeter.Init(...);
#endif
```

## 六、注释规范

### 6.1 Doxygen（公开接口）

```cpp
/**
 * @brief 优劣弧判断：对比当前角度与目标角度，必要时翻转目标
 * @param target 目标角度，若走劣弧则修改为翻转后的值
 * @return 方向因子：1.0 正转 / -1.0 反转
 */
static float OptimalArc(float current, float& target);
```

### 6.2 单行注释

```cpp
// 解释"为什么"，不解释"是什么"
// 走短路径：目标角度 ±π，行进电机反转
target = NormalizeAngle(current + err - (err > 0.f ? kPi : -kPi));
```

### 6.3 尾部注释

对齐到同一列，说明变量用途：

```cpp
static constexpr float kTorqueK = 0.3f;  // C6xx 转矩常数 N·m/A
static float g_vx = 0.0f;                // 底盘速度指令 X
```

### 6.4 常量必须标注来源

关键数值必须说明来源，否则 3 个月后无人能改：

```cpp
constexpr int16_t kMax = 1684;          // DR16 协议标称值
constexpr float k1Init = 1.453e-7f;     // 拟合数据
constexpr float kTorqueK = 0.3f;        // 手册数据
```

## 七、线程安全

### 7.1 ISR → 线程：seqlock

```cpp
// 写入（中断上下文）
void Callback(uint8_t* buf) {
    atomic_inc(&seq_);          // 开始：seq 变为奇数
    field1_ = ...;
    field2_ = ...;
    atomic_inc(&seq_);          // 结束：seq 变为偶数且递增
}

// 读取（控制线程）
Snapshot ReadAll() const {
    Snapshot s;
    atomic_t seq;
    do {
        seq = atomic_get(&seq_);
        if (seq & 1) continue;
        s.a = field1_;
        s.b = field2_;
    } while (atomic_get(&seq_) != seq);
    return s;
}
```

### 7.2 单字段原子访问

32-bit RISC-V 上 aligned float 读写是天然的原子操作，单字段 getter 不需要 seqlock。

### 7.3 线程间通信

```
ISR → seqlock → 控制线程（同一模块内）
控制层 → zbus_pub → CAN 发送线程（跨模块）
```

## 八、核心原则

### 8.1 动手前读全文件

改代码前，把相关文件全部读完。不要只看一个文件就下结论。不要假设两个文件矛盾——它们可能有各自的用途。

### 8.2 只做用户要求的事

用户说"加个功能"，就只加功能。不提前重构、不改命名、不删类、不移动头文件。

### 8.3 不猜数值

代码里的数值不是猜出来的。看协议文档或问清楚。

### 8.4 验证改动能编译

改完至少确认能编译通过。不要等 CI/build 炸了才回头查。

### 8.5 不留垃圾

旧文件及时删。多余空目录不要留。注释掉的代码要么删掉（git history 里有），要么加说明"待实现"。

## 九、常见反模式

| 反模式 | 正确做法 |
|--------|---------|
| 头文件中 `using namespace` | cpp 的具名 namespace 内部使用 |
| 魔法数字硬编码 | `constexpr` 命名常量 + 来源注释 |
| 全局变量跨线程 | zbus 发布订阅 |
| 固定 `k_msleep` 控制周期 | 测量耗时 + 动态补偿 |
| `new[]` / `delete[]` 动态分配 | 编译期确定大小，模板或静态数组 |
| 大量复制粘贴雷同代码 | 提取公共函数/模板 |
| 头文件缺 `#pragma once` | 每个 hpp/h 文件都必须有 |
