# 功率控制 — 采样、逻辑与计算流程详解

> **说明**：本文档以 `Power_Ctrl()` 函数为主线，配合 `Control_Task` 主循环，逐段解释功率控制从数据采样→模型预测→功率分配→受限求解→输出替换的完整流程。

---

## 一、整体调用关系

### 1.1 Control_Task 主循环

功率控制位于底盘 `Control_Task` 1ms 周期的**最末端**。完整顺序如下：

```c
void Control_Task(void const *argument) {
    Control_Init(&Chassis_Info);
    for (;;) {
        // 1. 模式选择
        Control_Mode(&Chassis_Info);

        // 2. 舵轮校准（接近开关边沿捕获+PID归零）
        Chassis_Cal(&Chassis_Info);

        // 3. 读取电机/云台测量值
        Control_Measure_Update(&Chassis_Info);

        // 4. 运动学解算 + 目标值计算
        Control_Target_Update(&Chassis_Info);

        // 5. 热量控制（射速调节）
        Fire_Ctrl();

        // 6. 卡弹检测与反转解除
        Stop_Detection();

        // 7. 复活校准检测（断电解锁重校准）
        Revival_Calibration();

        // 8. PID 核心计算
        Control(&Chassis_Info);
        //   ├─ 云台Yaw串级PID (键盘/视觉/自动)
        //   ├─ Buffer能量PID
        //   ├─ 4×舵向角度PID (位置环)
        //   ├─ 4×舵向速度PID (速度环)
        //   └─ 4×行进速度PID

        // 9. PID输出 → Chassis_Info.Output (原始值)
        Control_Info_Update(&Chassis_Info);

        // 10. 功率控制 (条件执行)
        if ((Chassis_Info.trigger.need_trigger == 2) &&
            (Referee_System_Info.game_status.game_progress == 4)) {
            Control_SuperCap_Mode_Update(&Control_Info);
            Power_Ctrl();   // ← 核心函数
        }

        osDelay(1);
    }
}
```

### 1.2 Power_Ctrl 执行条件

| 条件 | 含义 |
|------|------|
| `need_trigger == 2` | 舵轮校准已完成（4个舵轮均已找到零位） |
| `game_progress == 4` | 比赛正在进行 |
| 任一条件不满足 | 跳过功率控制，直接使用 PID 原始输出值 |

## 二、前置步骤：超级电容功率上限更新

`Power_Ctrl()` 前先调用的 `Control_SuperCap_Mode_Update()` 决定了本周期功率预算：

```c
void Control_SuperCap_Mode_Update(Control_Info_Typedef *Control_Info) {
    if (Chassis_Info.Supercap.Persent >= 15) {
        // 电容电量≥15%: 加100W缓冲
        Chassis_Info.Supercap.Power_Max = 
              Referee_System_Info.robot_status.chassis_power_limit 
            - PID_Buffer.Output 
            + 100;
    } else {
        // 电容电量不足: 仅使用裁判系统限制
        Chassis_Info.Supercap.Power_Max = 
              Referee_System_Info.robot_status.chassis_power_limit 
            - PID_Buffer.Output;
    }
    Chassis_Info.Power_Max = Chassis_Info.Supercap.Power_Max;
}
```

其中 `PID_Buffer.Output` 是能量buffer的PID控制输出（目标60J → 控制功率消耗速度）：

```c
static float PID_Buffer_Param[7] = {3, 0, 0, 0, 0, 0, 60};
// 在 Control() 中执行:
PID_Calculate(&PID_Buffer, 60, Referee_System_Info.power_heat_data.buffer_energy);
```

## 三、Power_Ctrl 完整分段详解

### Phase 1 — 电机数据采样（第861~874行）

循环读取8个电机的实时数据（4个舵向 + 4个行进）：

```
输入:  Motor_Course[i].Data.Current   (舵向电机电流, int16_t, 范围 -16384~16384)
       Motor_Course[i].Data.Velocity   (舵向电机转速, int16_t, RPM)
       Motor_Advance[i].Data.Current   (行进电机电流, int16_t)
       Motor_Advance[i].Data.Velocity   (行进电机转速, int16_t, RPM)

输出:  Omiga[i]     → 角速度 (rad/s)
       Torque[i]    → 转矩 (N·m)
       Torque_2[i]  → 转矩平方
       Omiga_2[i]   → 角速度平方
       Power_In[i]  → 单电机预测功率
```

**代码**：

```c
for (i = 0; i < 4; i++) {
    // ── ① 舵向电机 ──
    // 电流低通滤波 (Alpha=0.1, 滤除高频噪声)
    LowPassFilter1p_Update(&LPF_Motor[i], Motor_Course[i].Data.Current);

    // rpm → rad/s: ω = Velocity / 9.55  (9.55 = 60 / 2π)
    PowerCtrl_Info[0].Target.Omiga[i] = Motor_Course[i].Data.Velocity / 9.55;

    // 电流 → 转矩: τ = Current_filtered × 4.577e-5
    // 4.577e-5 是 M3508 电机转矩常数 (N·m/A)
    PowerCtrl_Info[0].Target.Torque[i] = LPF_Motor[i].Output * 4.577e-5;

    // 平方项 (用于RLS特征和功率模型)
    PowerCtrl_Info[0].Target.Omiga_2[i]  = Omiga[i]²;
    PowerCtrl_Info[0].Target.Torque_2[i] = Torque[i]²;

    // ── ② 行进电机 ── (同上, 但电流不过LPF)
    PowerCtrl_Info[1].Target.Omiga[i]  = Motor_Advance[i].Data.Velocity / 9.55;
    PowerCtrl_Info[1].Target.Torque[i] = Motor_Advance[i].Data.Current * 4.577e-5;
    PowerCtrl_Info[1].Target.Omiga_2[i]  = Omiga[i]²;
    PowerCtrl_Info[1].Target.Torque_2[i] = Torque[i]²;
}
```

**为什么舵向电流要加 LPF，行进电流不用？**

舵向电机（M3508角度轮）在保持角度时电流波动大，LPF 减少噪声对功率预测的影响。行进电机直接电流反馈，响应更快。

### Phase 2 — 单电机功率预测（第867~873行）

模型预测**每个电机**当前的消耗功率：

```c
// 舵向电机[i] 的预测功率
Power_In[i] = K1 × τ² + K2 × ω² + τ × ω
// 注意: K3 不在逐电机层面加，在求和阶段统一加

// 行进电机[i] 的预测功率 (同理)
Power_In[i] = K1 × τ² + K2 × ω² + τ × ω
```

**模型物理含义**：

| 项 | 物理意义 | 对应损耗 |
|----|----------|----------|
| `K1 × τ²` | 铜损（电流热效应） | 与电流平方成正比 |
| `K2 × ω²` | 铁损/摩擦损（涡流+轴承摩擦） | 与转速平方成正比 |
| `τ × ω` | 机械功率输出 | 做有用功的部分 |
| `K3` | 静止损耗 | 电路基础功耗 |

### Phase 3 — 求和与功率预算（第876~889行）

**③-a 组内求和工作**：

```c
// 舵向组求和
Sum.Torque2_Sum = τ²[0] + τ²[1] + τ²[2] + τ²[3];   // RLS特征1
Sum.Omiga2_Sum  = ω²[0] + ω²[1] + ω²[2] + ω²[3];     // RLS特征2
Sum.Power_Sum   = P_in[0]+P_in[1]+P_in[2]+P_in[3]+K3; // 总预测功率+常数
Power_Allin     = Sum.Power_Sum;                       // 组预测总功率

// 行进组求和 (同理)
Sum.Torque2_Sum = τ²[0] + τ²[1] + τ²[2] + τ²[3];   // RLS特征3
Sum.Omiga2_Sum  = ω²[0] + ω²[1] + ω²[2] + ω²[3];     // RLS特征4
Sum.Power_Sum   = P_in[0]+P_in[1]+P_in[2]+P_in[3]+K3;
Power_Allin     = Sum.Power_Sum;
```

**③-b 功率预算分配**：

```c
// ── 舵向: 最多拿 80% ──
PowerCtrl_Info[0].Power_Max = 0.8 × Chassis_Info.Power_Max;
VAL_LIMIT(Power_Allin, -Power_Max, Power_Max);  // 限幅

// ── 行进: 拿剩余部分（动态变化）──
PowerCtrl_Info[1].Power_Max = Chassis_Info.Power_Max - |PowerCtrl_Info[0].Power_Allin|;
VAL_LIMIT(Power_Allin, -Power_Max, Power_Max);
```

**关键理解**：
- `Power_Max[0] = 80%` 是**硬上限**，舵向预测功率永远不会超过这个值
- `Power_Max[1]` = 总功率 − **舵向实际消耗**，舵向用得多行进就少，反之亦然
- 如果舵向只用了 10%，行进就能拿 90%

### Phase 4 — RLS 数据装配（第890~902行）

将数据装配给 RLS 用于在线参数辨识（当前实际被注释）：

```c
// 实测功率低通滤波 (Alpha=0.7)
LowPassFilter1p_Update(&LPF_ChassisPower, Chassis_Info.Supercap.Chassis_Power);

// 模型输出 u = 预测总功率
RLS_Power_Info.Data.U[0] = PowerCtrl_Info[0].Power_Allin 
                          + PowerCtrl_Info[1].Power_Allin;

// 观测值 y = 实测功率
RLS_Power_Info.Data.Y[0] = LPF_ChassisPower.Output;

// 特征向量 X (4维):
RLS_Power_Info.Data.X[0] = PowerCtrl_Info[0].Sum.Torque2_Sum;  // 舵向∑τ²
RLS_Power_Info.Data.X[1] = PowerCtrl_Info[0].Sum.Omiga2_Sum;   // 舵向∑ω²
RLS_Power_Info.Data.X[2] = PowerCtrl_Info[1].Sum.Torque2_Sum;  // 行进∑τ²
RLS_Power_Info.Data.X[3] = PowerCtrl_Info[1].Sum.Omiga2_Sum;   // 行进∑ω²

// RLS_Update(&RLS_Power_Info); ← 在线更新 (当前被注释)

// 提取参数 (无论是否RLS更新, 始终从 W 数组读取)
PowerCtrl_Info[0].K1 = RLS_Power_Info.Data.W[0];
PowerCtrl_Info[0].K2 = RLS_Power_Info.Data.W[1];
PowerCtrl_Info[1].K1 = RLS_Power_Info.Data.W[2];
PowerCtrl_Info[1].K2 = RLS_Power_Info.Data.W[3];
```

**如果取消注释 `RLS_Update()`**，每周期会执行完整的递归最小二乘更新：

```
① K(n) = P(n-1)·X(n) / [λ + Xᵀ(n)·P(n-1)·X(n)]   ← 计算卡尔曼增益
② Ŵ(n) = Ŵ(n-1) + K(n)·[y(n) - u(n)]                ← 更新参数
③ P(n) = [P(n-1) - K(n)·Xᵀ(n)·P(n-1)] / λ            ← 更新协方差
```

其中 λ=0.99999（遗忘因子，接近1表示历史数据权重衰减极慢）。

### Phase 5 — 隶属度计算与功率分配（第904~939行）

这是功率控制中最巧妙的部分。它解决了一个核心矛盾：**总功率不够分时，如何决定哪个电机该被限？**

**⑤-a 读取PID误差**：

```c
// 舵向: 角度速度环的误差
PowerCtrl_Info[0].Err[i] = PID_Chassis_angle_v[i].Err[0];
// 行进: 速度环的误差
PowerCtrl_Info[1].Err[i] = PID_Chassis_velocity[i].Err[0];
```

**⑤-b 计算权重 K**（模糊逻辑中的"隶属度权重"）：

```c
// 舵向
if (∑|Err| > 50)   K = 1;           // 误差大 → 按需求分配
else if (∑|Err| < 0.01)  K = 0;    // 误差小 → 按现状分配
else K = (∑ - 0.01) / (50 - 0.01);  // 中间态线性插值

// 行进
if (∑|Err| > 500)  K = 1;
else if (∑|Err| < 0.001) K = 0;
else K = (∑ - 0.001) / (500 - 0.001);
```

**K 的含义**:

| K=0 | K=1 | K=0.5 |
|-----|-----|-------|
| 误差很小，不需要额外干预 | 误差很大，急需纠正 | 均衡状态 |
| 按当前功率占比分配 | 按误差比例分配 | 各占一半权重 |

**⑤-c 计算单电机隶属度与功率上限**：

```c
for (i = 0; i < 4; i++) {
    Membership[i] = K × (|Err[i]| / ∑|Err|)           // "需要多少"
                  + (1-K) × (|P_in[i]| / ∑|P_in|);  // "正在用多少"
    VAL_LIMIT(Membership[i], 0, 1);

    Power_Limit[i] = Membership[i] × Power_Max;
    // 这个电机本周期最多能用 Power_Limit 瓦
}
```

**一个具体例子**：

假设底盘功率上限 45W，舵向 `Power_Max = 36W`（80%）。4个电机当前状态：

| 电机 | 预测功率 | PID误差 | ∑误差=60 (K=1) | ∑误差=5 (K=0.1) |
|------|----------|---------|----------------|----------------|
| FL | 15W | 30 | 30/60×36=**18W** | 0.1×.5+.9×.42→**16W** |
| FR | 8W | 10 | 10/60×36=**6W** | 0.1×.17+.9×.22→**8W** |
| RL | 7W | 12 | 12/60×36=**7W** | 0.1×.2+.9×.19→**7W** |
| RR | 6W | 8 | 8/60×36=**5W** | 0.1×.13+.9×.17→**5W** |

→ 误差大时（K=1），误差最大的 FL 拿到最多功率 → **快速收敛**
→ 误差小时（K→0），各电机功率比例 ≈ 当前消耗比例 → **稳定运行**

### Phase 6 — 舵向受限力矩求解（第941~976行）

**判断是否超功率**：

```c
if (Sum.Power_Sum >= Power_Max) {
    // 预测总功率超过分配上限 → 需要限制
```

**逐电机解二次方程**：

从功率模型反推允许的最大转矩：

```
P_limit = K1 × τ² + ω × τ + K2 × ω² + K3

整理为:
K1 × τ² + ω × τ + (K2 × ω² + K3 - P_limit) = 0

即:  A × τ² + B × τ + C = 0
其中:
  A = K1
  B = ω          (已知, 当前角速度)
  C = K2×ω² + K3 - Power_Limit[i]
```

**代码实现**：

```c
for (i = 0; i < 4; i++) {
    // 构建二次方程
    A = PowerCtrl_Info[0].K1;   // 铜损系数
    B = PowerCtrl_Info[0].Target.Omiga[i];  // 当前角速度
    C = K2 × ω² + K3 - Power_Limit[i];      // 剩余功率预算

    Δ = B² - 4 × A × C;

    // 保护: Δ 无效时强制为0
    if (isnan(Δ) || isinf(Δ) || Δ < 0) Δ = 0;

    if (Δ >= 0) {
        τ = (-B + √Δ) / (2 × A);  // 取正根
        // 转矩 → 电流值 (替换PID输出)
        Chassis_Info.Output.Chassis_Angel[i] = τ / 4.577e-5;
    }
}
```

**重点理解**：
- `Power_Limit[i]` 是 Phase 5 为这个电机算出的功率上限
- 方程解出的 τ 是**满足功率限制的最大允许转矩**
- 计算的是转矩→换算成电流→直接替换 `Chassis_Info.Output`
- 无论正反转，都取 `+√Δ` 的根（因为 `B=ω` 的正负已经包含了方向；有符号时另一个根物理上无意义）

**如果没有超功率**：

```c
else {
    // 直接用 PID 计算的原值
    Chassis_Info.Output.Chassis_Angel[i] = PID_Chassis_angle_v[i].Output;
}
```

### Phase 7 — 行进受限力矩求解（第978~1020行）

与舵向结构完全一致，但有两点细微差异：

```c
if (PowerCtrl_Info[1].Sum.Power_Sum >= PowerCtrl_Info[1].Power_Max) {
    for (i = 0; i < 4; i++) {
        A = PowerCtrl_Info[1].K1;
        B = PowerCtrl_Info[1].Target.Omiga[i];  // 行进角速度
        C = K2×ω² + K3 - Power_Limit[i];
        Δ = B² - 4×A×C;

        if (Δ >= 0) {
            // ★ 差异: 根据输出方向选择根
            if (Chassis_Info.Output.Chassis_Velocity[i] >= 0)
                τ = (-B + √Δ) / (2×A);
            else
                τ = (-B - √Δ) / (2×A);

            Chassis_Info.Output.Chassis_Velocity[i] = τ / 4.577e-5;
        }
    }
}
```

**为什么行进需要判断正负根？**

行进电机的目标速度可正可负（前进/后退）。正输出时取 `+√Δ`、负输出时取 `-√Δ`，确保：
- 限制后的力矩**方向不变**
- 限制后的力矩**幅值不超**

而舵向电机的角度环输出始终在 ±180° 范围内，正负由角度偏差的方向决定，统一取 `+√Δ` 即可。

else {

## 四、完整时序图

```
时间 (1ms)
│
├─ Control_Measure_Update
│   ├─ Motor_Course[0..3].Current/Velocity → 舵向原始数据
│   └─ Motor_Advance[0..3].Current/Velocity → 行进原始数据
│
├─ Control_Target_Update
│   ├─ Vx, Vy, Vw → 运动学解算 → Target_Angle[4], Target_Velocity[4]
│   └─ 优劣弧判断 → K[4] 因子
│
├─ Control (PID计算)
│   ├─ 4× 舵向角度PID (位置环)
│   ├─ 4× 舵向速度PID (速度环) → PID_Chassis_angle_v[i].Output  (原始值A)
│   └─ 4× 行进速度PID           → PID_Chassis_velocity[i].Output  (原始值V)
│
├─ Control_Info_Update
│   └─ Chassis_Info.Output.Chassis_Angel[i]    = 原始值A
│      Chassis_Info.Output.Chassis_Velocity[i]  = 原始值V
│
└─ Power_Ctrl()
    │
    ├─ [采样] 读取8个电机的Current/Velocity
    │   └─ LowPassFilter → Torque, Omega
    │
    ├─ [预测] 8× P_in = K1τ² + K2ω² + τω
    │
    ├─ [预算] 舵向Max=80%, 行进Max=剩余
    │
    ├─ [RLS]  装配X=[∑τ₁²,∑ω₁²,∑τ₂²,∑ω₂²], y=P_实测 (当前跳过更新)
    │
    ├─ [隶属度] 读PID误差 → 计算K → Membership[i] → Power_Limit[i]
    │
    ├─ [舵向限制] if 超功率 → 解方程 τ → 替换 Output.Chassis_Angel[i]
    │              不超    → 保持原始值A
    │
    └─ [行进限制] if 超功率 → 解方程 τ → 替换 Output.Chassis_Velocity[i]
                   不超    → 保持原始值V

下一个1ms周期 → CAN_Task 发送 Output[8] 到电机驱动器
```

## 六、特殊情况处理

### 6.1 校准未完成（need_trigger ≠ 2）

跳过 `Power_Ctrl()`，直接发送 PID 原始值。此时功率不受限，适合调试。

### 6.2 比赛未开始（game_progress ≠ 4）

同上，跳过功率控制。

### 6.3 Off 模式

在 `Control_Info_Update()` 中就将所有输出置零，`Power_Ctrl()` 不会执行：

```c
if (Control_Info.Control_Mode == Off) {
    Vel[4] = 0; Ang[4] = 0; Yaw=0; Shoot=0;
    return;  // 直接返回，不执行后面的功率控制
}
```

### 6.4 NaN/Inf 保护

二次方程判别式 Δ 若为 NaN、Inf 或负数，强制置零：

```c
if (isnan(Delta) == 1 || isinf(Delta) == 1 || Delta < 0)
    Delta = 0;
```

确保限制后的输出不会产生非法值。
