# 舵轮（Swerve Drive）逻辑控制算法详解

> **硬件配置**：4个M3508角度电机（舵向）+ 4个M3508行进电机  
> **传感器**：洛施达微型接近开关（零位校准）  
> **控制模式**：Follow / Vision / Auto / Spin 等8种模式

---

## 一、舵轮底盘概述

### 1.1 什么是舵轮（Swerve Drive）

四轮独立舵轮结构，每个轮子有两个自由度：
- **舵向电机 (Course/Steer)**：控制轮子朝向（±180°），使用 M3508 + C610 驱动器
- **行进电机 (Advance/Drive)**：控制轮子转动速度，使用 M3508 + C620 驱动器

这使得底盘能够**全向移动**，在任意方向平移的同时自转，非常适合 RoboMaster 哨兵需要灵活调整攻防角度的场景。

### 1.2 整体控制架构

```
┌──────────────────────────────────────────────────────────────────┐
│                   舵轮底盘控制数据流                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                    │
│  Vx, Vy, Vw (运动指令)                                            │
│         │                                                         │
│         ▼                                                         │
│  ┌─────────────────────────┐                                      │
│  │   舵轮运动学解算         │  ← 核心: 反向解算4个轮子的目标角度和速度  │
│  │   Mecanum-like IK       │                                      │
│  └─────────┬───────────────┘                                      │
│            │                                                      │
│  ┌─────────▼───────────────┐                                      │
│  │  优劣弧判断             │  ← 确保角度最优路径（< 90°）            │
│  │  Optimal Arc Decision   │                                      │
│  └─────────┬───────────────┘                                      │
│            │                                                      │
│  ┌─────────▼───────────────┐                                      │
│  │  串级PID控制器          │  ← 角度环 → 速度环                     │
│  │  Cascade PID            │                                      │
│  └─────────┬───────────────┘                                      │
│            │                                                      │
│  ┌─────────▼───────────────┐                                      │
│  │  功率限制模块            │  ← Power_Ctrl(), 约束最大电流           │
│  │  Power Limiter          │                                      │
│  └─────────┬───────────────┘                                      │
│            │                                                      │
│            ▼                                                      │
│      CAN总线 → C610/C620 → M3508电机                              │
└──────────────────────────────────────────────────────────────────┘
```

## 二、舵轮运动学解算

### 2.1 坐标系定义

```
        车头方向 (0°)
              ↑
              |
    FL ───────┼─────── FR
              |  R
    ←─────────┼─────────→ X (Vy)
              |
    RL ───────┼─────── RR
              |
              ↓ Y (Vx)
```

- **Vx**：前后方向速度（+向前）
- **Vy**：左右方向速度（+向右）
- **Vw**：自转角速度（+逆时针）
- **R**：轮子到底盘中心的距离（0.135m）
- **sin45**：√2/2 ≈ 0.7071

### 2.2 舵向角度解算

每个轮子的目标舵向角由运动指令 Vx, Vy, Vw 通过 atan2 计算：

```c
#define R     0.135      // 轮子到底盘中心距离 (m)
#define sin45 0.70710678 // √2/2

// FL (前左)
Chassis_Info->Target.Chassis_Angle[0] = 
    -(atan2f((Vx + Vw * sin45 * R), (Vy + Vw * sin45 * R)) * RadiansToDegrees);
// FR (前右)
Chassis_Info->Target.Chassis_Angle[1] = 
    -(atan2f((Vx + Vw * sin45 * R), (Vy - Vw * sin45 * R)) * RadiansToDegrees);
// RL (后左)
Chassis_Info->Target.Chassis_Angle[2] = 
    -(atan2f((Vx - Vw * sin45 * R), (Vy + Vw * sin45 * R)) * RadiansToDegrees);
// RR (后右)
Chassis_Info->Target.Chassis_Angle[3] = 
    -(atan2f((Vx - Vw * sin45 * R), (Vy - Vw * sin45 * R)) * RadiansToDegrees);
```

**原理推导**：每个轮子的速度向量为 `V_vehicle + ω × r`，atan2 求该向量与 X 轴之间的夹角即为舵向角。`θ = -atan2(V_x_component, V_y_component)` ，负号是因为坐标系方向差异。

### 2.3 优劣弧判断 (Optimal Arc)

由于舵向电机的范围在 ±180°，如果目标角度与当前角度差超过 90°，旋转 >90° 不如**反向旋转**然后 **行进电机反转** 更高效。

**判断逻辑**：
```c
Absolute_Angle[i] = Calibration_Offset[i] - Motor_Current_Angle + Target_Angle[i];

if (fabs(Absolute_Angle[i]) < 90.f) {
    K[i] = 1.0f;    // 正向最优
}
else if (Absolute_Angle[i] >= 90) {
    Absolute_Angle[i] -= 180.0f;
    K[i] = -1.0f;    // 反向最优，行进电机反转
}
else if (Absolute_Angle[i] <= -90) {
    Absolute_Angle[i] += 180.0f;
    K[i] = -1.0f;    // 反向最优，行进电机反转
}
```

其中 `K[i]` 是行进速度方向因子：
- `K = 1`：行进电机正转
- `K = -1`：行进电机反转（舵向绕了180°，行进方向反过来补偿）

### 2.4 行进速度解算

根据舵向因子 K 修正后的行进速度：

```c
// FL
Chassis_Velocity[0] = -K[0] * √[(Vx + Vw*sin45*R)² + (Vy + Vw*sin45*R)²];
// FR
Chassis_Velocity[1] = -K[1] * √[(Vx + Vw*sin45*R)² + (Vy - Vw*sin45*R)²];
// RL
Chassis_Velocity[2] =  K[2] * √[(Vx - Vw*sin45*R)² + (Vy + Vw*sin45*R)²];
// RR
Chassis_Velocity[3] =  K[3] * √[(Vx + Vw*sin45*R)² + (Vy - Vw*sin45*R)²];
```

速度正负号由轮子布局几何决定（FL/FR 为负是因为 M3508 的安装方向）。

## 三、舵轮校准（零位标定）

### 3.1 为什么需要校准

M3508 电机使用增量式编码器，**断电后角度归零**，每次上电时舵向的机械零位与编码器零位不一致。必须通过外部传感器（接近开关）确定零位。

### 3.2 接近开关

每个舵轮安装一个接近开关（洛施达微小型方形接近开关 82.5NOF）：
- GPIO 触发，检测舵轮是否到达机械零位
- 每个舵轮对应一个 GPIO 引脚：

| 引脚 | 轮位 |
|------|------|
| GPIOE_PIN_13 | FL |
| GPIOE_PIN_9 | FR |
| GPIOA_PIN_2 | RL |
| GPIOA_PIN_0 | RR |

### 3.3 校准状态机

```
                    ┌─────────┐
                    │ 未校准   │
                    │ Grab=0   │
                    └────┬────┘
                         │ 检测到接近开关信号
                         ▼
                    ┌─────────┐
                    │ 首次捕获  │  ← 记录下降沿角度 Angle_now
                    │ Grab[0]=1│
                    └────┬────┘
                         │ 接近开关释放
                         ▼
                    ┌─────────┐
                    │ 二次捕获  │  ← 记录上升沿角度 Angle_last
                    │ Grab[1]=1│     计算零位 = (now + last)/2
                    └────┬────┘
                         │ PID闭环回归零位
                         ▼
                    ┌─────────┐
                    │ 校准完成  │
                    │ Cal=1   │
                    └─────────┘
```

### 3.4 校准过程详解

**Step 1 — 边沿捕获**（GPIO中断回调）：

```c
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin) {
    // 下降沿: 检测到接近开关 (金属接近)
    if (GPIO_Pin == GPIO_PIN_13 && now_state == 0 && last_state == 1) {
        Grab[0][FL] += 1;
        Chassis_Angle_now[FL] = Motor_Course[FL].Data.Angle;
    }
    // 上升沿: 接近开关释放
    if (GPIO_Pin == GPIO_PIN_13 && now_state == 1 && last_state == 0) {
        Grab[1][FL] += 1;
        Chassis_Angle_last[FL] = Motor_Course[FL].Data.Angle;
        // 零位 = 下降沿角度与上升沿角度的平均值
        Chassis_Angle_average[0][FL] = (Chassis_Angle_now[FL] + Chassis_Angle_last[FL]) * 0.5;
    }
}
```

**Step 2 — PID闭环归零**（Chassis_Cal 函数）：

```c
for (i = 0; i < 4; i++) {
    Error[i] = Cal_Zero[i] - Motor_Course[i].Data.Angle;
    
    if (Grab[0][i] > 1 && Grab[1][i] > 1) {
        // 已捕获完整的接近开关信号
        // 更新零位参考值 (再次平均并偏移 -3° 确保在检测区域内)
        Chassis_Angle_average[1][i] = (now + last) * 0.5;
        if (need_trigger == 1) {
            Chassis_Angle_average[2][i] = 
                ((平均0 + 平均1) * 0.5) - 3.0f;  // 偏移3°确保稳定
        }
        // PID闭环归零: 角度环 + 速度环
        PID_Calculate(&PID_Chassis_Cal[i], 参考零位, 当前角度);
        PID_Calculate(&PID_Chassis_Cal_V[i], PID_Cal[i].Output, 当前速度);
        Calibrate_State[i] = 1;  // 标记校准完成
    }
    else {
        // 未捕获: 强制旋转寻找接近开关
        if (Grab[1][i] < 1) 目标角度 = 100°;   // 正向搜索
        if (Grab[1][i] == 1) 目标角度 = -90°;  // 已捕获一次，反向搜索
    }
}
```

### 3.5 复活校准

如果底盘掉电或机器人被裁判系统关闭底盘输出，重新上电时需要重新校准：

```c
static void Revival_Calibration(void) {
    if (底盘输出断电 || 当前血量 == 0) {
        Revival_Calibration_Grap = 1;
    }
    if (Control_Mode == Off) {
        // 重置所有校准状态
        need_trigger = 1;           // 进入校准模式
        Grab[0][i] = 0;            // 清空捕获计数
        Grab[1][i] = 0;
        Calibrate_State[i] = 0;     // 校准未完成
        Chassis_Angle[i] = 0.0f;
        Power_Max = 100;            // 恢复默认功率
    }
}
```

## 四、串级PID控制

### 4.1 控制架构

每个舵轮采用**三环/两环串级控制**：

```
舵向控制链:
  舵向角度环 (P=62) → 舵向速度环 (P=20) → [功率限制] → C610 → M3508

行进控制链:
  行进速度环 (P=15) → [功率限制] → C620 → M3508
```

### 4.2 PID 参数

| 控制器 | P | I | D | I_MAX | Output_MAX | 用途 |
|--------|---|---|---|-------|------------|------|
| `PID_Chassis_angle[i]` | 62 | 0 | 0 | 1 | 8000 | 舵向角度环 |
| `PID_Chassis_angle_v[i]` | 20 | 0 | 0 | 10 | 8000 | 舵向速度环 |
| `PID_Chassis_velocity[i]` | 15 | 0 | 0 | 1000 | 10000 | 行进速度环 |

> 注意：所有舵轮 PID 均为纯 P 控制（I=0, D=0），通过调试保证了足够的响应速度。

### 4.3 云台跟随PID

底盘需要根据云台偏航角进行跟随旋转，有两种模式：

```c
// 键盘跟随 (Follow模式)
Follow_PID[0] = {P=8, D=0.1}    // 角度环：目标=0, 反馈=Angle_gap
Follow_PID[1] = {P=-24}          // 速度环

// 视觉自瞄 (Vision模式)
PID_Yaw_Vision[0] = {P=-35}      // 角度环
PID_Yaw_Vision[1] = {P=205, I=0.2, D=2, Alpha=0.5}  // 速度环 + 微分LPF

// 自动导航 (Auto模式)
PID_Auto = {P=250, D=0.1}        // 速度环 (控制云台yaw陀螺仪)
```

### 4.4 微分前馈补偿

在 Vision 模式下，使用前馈补偿提高跟踪速度：

```c
// 计算PID输出变化量 × 增益系数作为前馈
aba = (PID_Yaw_Vision[0].Output - Last_Yaw_PID_Output) * 1.4;

// 角度环输出 - 前馈量 → 速度环
PID_Calculate(&PID_Yaw_Vision[1], PID_Yaw_Vision[0].Output - aba, Gyro);
```

## 五、各模式控制逻辑

### 5.1 模式总览

```
┌──────────────────────────────────────────────────────────┐
│                    模式切换状态机                           │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  遥控器 SW1=1, SW2=1 → Auto (全自动)                      │
│  遥控器 SW1=1, SW2=2 → Chassis_Auto (底盘自动)            │
│  遥控器 SW1=1, SW2=3 → Spin_Vision (视觉自旋)             │
│  遥控器 SW1=2, SW2=1 → Shoot (射击)                       │
│  遥控器 SW1=2, SW2=2 → Off (关闭)                         │
│  遥控器 SW1=2, SW2=3 → Vision (视觉跟随)                  │
│  遥控器 SW1=3, SW2=1 → Spin (旋转)                        │
│  遥控器 SW1=3, SW2=2 → Chassis_Off (底盘关闭)             │
│  遥控器 SW1=3, SW2=3 → Follow (跟随)                      │
│                                                           │
│  遥控器丢失(rc_lost=1)      → Off                         │
│  遥控器丢失+比赛中(game=4)  → Auto (自动哨兵模式)          │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

### 5.2 Follow 跟随模式

底盘跟随云台偏航角，维持车头朝向云台指向：

```c
case Follow:
    // Angle_gap = 云台偏航角 - 底盘前向基准角
    if (fabs(Angle_gap) < 0.05)
        Vw = 0;  // 死区
    else {
        // 串级PID计算自转速度
        Follow_PID[0] 角度环: err = 0 - Angle_gap
        Follow_PID[1] 速度环: Vw = PID(角度环输出, Yaw陀螺仪)
    }
    // Vx, Vy 由遥控器摇杆直接控制
    Vx = REMOTO_CTRL.rc.ch[3] * 5.0f;
    Vy = REMOTO_CTRL.rc.ch[2] * 5.0f;
break;
```

### 5.3 Vision 视觉模式

接收 MiniPC 视觉数据，实现自瞄跟踪：

```c
case Vision:
    // 目标偏航角来自视觉或手动微调
    if (视觉锁定)
        Target_Yaw = MiniPC_Vision_Yaw;
    else
        Target_Yaw += 遥控器.ch[0] * 0.0005;
    
    // 射击指令
    if (开火标志 && 遥控器扳机按下)
        摩擦轮转速 = 拨弹盘速度 × 540;
    else
        摩擦轮转速 = 0;
break;
```

### 5.4 Spin 旋转模式

固定角速度旋转，用于防御或补给区：

```c
case Spin:
    Vw = 2000.0f;   // 固定旋转速度
break;

case Spin_Vision:
    Vw = 2000.0f;   // 旋转
    // 同时叠加视觉跟踪偏航
    Target_Yaw = MiniPC_Vision_Yaw;  // 旋转中调整云台指向目标
break;
```

### 5.5 Auto 自动模式

哨兵全自动导航，结合视觉和导航组数据：

```c
case Auto:
case Chassis_Auto:
    // MiniPC数据转换为运动速度
    _Vy = -MiniPc_Receive_Auto.Vy * 2233.866;
    _Vx = MiniPc_Receive_Auto.Vx * 2233.866;
    
    // 坐标系旋转（Vx,Vy旋转Angle_gap）
    Control_Info.Move.Vx = -sin(Angle_gap) * _Vx + cos(Angle_gap) * _Vy;
    Control_Info.Move.Vy = cos(Angle_gap) * _Vx + sin(Angle_gap) * _Vy;
    
    // 自转角速度: 静止时交替变换(3000↔5000), 运动时固定5000
    if (|_Vx| < 35 && |_Vy| < 35) {
        // 静止时 Vw 在 3000 和 5000 之间交替
        target_vw = is_odd_second ? 5000 : 3000;
    } else {
        target_vw = 5000;  // 移动中固定速度
    }
break;
```

### 5.6 坐标系旋转

在 Auto 模式中，MiniPC 给出的 Vx, Vy 是**以云台朝向为基准**的，需要旋转到底盘坐标系：

```c
// 底盘坐标系 = 云台坐标系 × 旋转矩阵
Vx_chassis = -sin(Angle_gap) × Vx_gimbal + cos(Angle_gap) × Vy_gimbal;
Vy_chassis =  cos(Angle_gap) × Vx_gimbal + sin(Angle_gap) × Vy_gimbal;
```

其中 `Angle_gap = 云台偏航角 - 底盘前向基准角（YAW电机的-173.1°偏移）`。

## 六、舵向角度 PID 控制特性

### 6.1 角度误差环形处理

由于舵向角度范围是 -180° ~ +180°，误差计算需要处理环绕：

```c
// 误差限幅在 ±180 内
if (Err >= 180)  Err -= 360;
if (Err <= -180) Err += 360;
```

### 6.2 绝对角度计算

```c
Absolute_Angle[i] = Calibration_Offset - Current_Motor_Angle + Target_Angle[i];
```

- `Calibration_Offset`：通过接近开关标定的零位偏移
- `Current_Motor_Angle`：M3508编码器当前的角度（0~360°）
- `Target_Angle[i]`：运动学解算的目标舵向角

这个计算将**电机编码器角度**映射到**底盘坐标系中的绝对角度**。

### 6.3 输出处理

每个控制周期，PID输出通过功率限制模块后，最终发送到CAN总线：

```c
// Control_Info_Update() 中的输出分配
switch (Control_Mode) {
    case Vision:
    case Auto:
    case Shoot:
    case Follow:
        // 4个舵向角度值
        Chassis_Info->Output.Chassis_Angel[i] = PID_Chassis_angle_v[i].Output;
        // 4个行进速度值
        Chassis_Info->Output.Chassis_Velocity[i] = PID_Chassis_velocity[i].Output;
        break;
    case Off:
        // 全部置零
        Chassis_Info->Output.Chassis_Velocity[i] = 0;
        Chassis_Info->Output.Chassis_Angel[i] = 0;
        break;
}
```

## 七、卡弹检测与处理

### 7.1 检测逻辑

```c
void Stop_Detection() {
    // 条件1: 电流超过9800且持续150个周期(约150ms)
    if (Shoot_Motor.Data.Current >= 9800 && !Shoot_Grap) {
        Stuck_Times++;
        if (Stuck_Times > 150) {
            Shoot_Grap = 1;  // 标记卡弹
        }
    }
    
    // 条件2: 速度 > 8000 但热量未变化 (弹丸未射出)
    if (速度 > 8000 && 热量不变 && 比赛中) {
        Stuck_Times++;
        if (Stuck_Times > 100) Stuck_Grap = 1;
    }
    
    // 处理: 反转拨弹盘
    if (Shoot_Grap == 1) {
        Return_Time++;
        输出 = PID_反向(-5400, 当前速度);  // 反转摩擦轮
        if (Return_Time >= 100) {
            if (电流 < 6000 || 超时 || 热量变化) {
                // 卡弹解除
                Return_Time = 0;
                Shoot_Grap = 0;
            }
        }
    }
    else {
        输出 = PID_正向(目标速度, 当前速度);
    }
}
```

### 7.2 反转解除

- 摩擦轮反转速度：5400 RPM（正转是6180~6400 RPM）
- 反转持续：100~150个控制周期（100~150ms）
- 解除条件：电流下降（<6000）或热量变化（弹丸已射出）或超时
