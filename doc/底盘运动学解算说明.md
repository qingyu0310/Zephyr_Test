# 底盘运动学解算说明

## 底盘布局

```
     A (舵轮)
  ┌──────┐
D │      │ B (从动轮)
  │      │
  └──────┘
     C (舵轮)
```

两个舵轮分别位于 A 边和 C 边（面对面），B、D 为从动轮。

- 前舵轮 (A 边) → **wi = 0**，位于 (0, +R)
- 后舵轮 (C 边) → **wi = 1**，位于 (0, -R)

## 算法流程

```
遥控器 axis.x / axis.y / axis.yaw
        │ (× 最大速度)
g_vx, g_vy, g_vw
        │
 SwerveKinCalculate()
        │
 左轮 target{angle, velocity}
 右轮 target{angle, velocity}
        │
 OptimalArc(当前底盘角度)
        │
 g_k_factor[2], g_steer_target[2]
        │
 motor[i] PID 计算
        │
 PowerCtrl → CAN TX
```

## 运动学解算 (`SwerveKinCalculate`)

### 速度分解

前轮 (y = +R)：
- `vx_front = g_vx - g_vw * R`     ← 旋转分量与运动方向相反
- `vy_front = g_vy`                ← Vy 不受旋转影响

后轮 (y = -R)：
- `vx_rear  = g_vx + g_vw * R`     ← 旋转分量与运动方向相同
- `vy_rear  = g_vy`                ← Vy 不受旋转影响

### 角度计算

```
angle = kAngleSign[wi] × atan2f(vx_w, vy_w)
```

- `atan2f(vx, vy)` 输出轮子指向速度方向的极角
- `kAngleSign[wi]` 调整舵向旋转方向

### 速度计算

```
velocity = hypot(vx_w, vy_w) × kSwerveSign{front/rear}
```

- `hypot` = 轮子线速度幅值
- `kSwerveSign{front/rear}` 调整行进电机正反转

## 优劣弧判断 (`OptimalArc`)

读转向电机当前角度，判断目标角度走"优弧"还是"劣弧"：

```
error = target - current
归一化到 ±π

如果 |error| > π/2:
    走劣弧：target += ±π（取短路径）
    k_factor = -1（行进电机反转）
否则：
    走优弧：直接转过去
    k_factor = 1（行进电机正转）
```

## 校准参数

底盘装好后，推摇杆观察轮子动作，通过修改以下常量来校准：

### 速度方向 (`kSwerveSignFront`, `kSwerveSignRear`)

| 现象 | 修改 |
|------|------|
| 推前后，轮子转向正确但车往反方向跑 | 对应轮的 Sign 取反（1 → -1 或 -1 → 1） |
| 推左右，轮子转向正确但车往反方向跑 | 对应轮的 Sign 取反 |

```
static constexpr int8_t kSwerveSignFront = -1;   // 前轮：-1 或 1
static constexpr int8_t kSwerveSignRear  = -1;   // 后轮：-1 或 1
```

### 角度方向 (`kAngleSign[2]`)

| 现象 | 修改 |
|------|------|
| 推前后，轮子不指向正前方（偏向左右） | 对应轮的 kAngleSign 取反 |
| 推左右，轮子不指向正左右方（偏前后） | 对应轮的 kAngleSign 取反 |

```
static constexpr int8_t kAngleSign[2] = {-1, -1};  // [前轮, 后轮]
```

### 综合判断

推前后摇杆（vy=0 纯 Vx），观察两轮：

| 现象 | 根因 | 解决 |
|------|------|------|
| 两轮指向正确（指向正前/正后），车倒退 | 速度方向反了 | `kSwerveSign` 取反 |
| 两轮指向正前/正后但反了（前轮指向后） | 角度方向反了 | `kAngleSign[wi]` 取反 |
| 一轮指向正确，一轮指向反了 | 该轮安装方向不同 | 单独改该轮的 Sign |
| 轮子一直在抖动或不稳定 | PID 参数或角度归一化问题 | 检查 PID，检查 NormalizeAngle |

## 坐标常量

| 常量 | 值 | 说明 |
|------|-----|------|
| `kSwerveR` | 0.135 m | 舵轮到底盘中心距离 |
| `KMaxMoveVelocity` | 2.0 m/s | 最大移动线速度 |
| `KMaxRotationOmega` | 2.0 rad/s | 最大旋转角速度 |

所有角度使用弧度，范围归一化到 ±π（`NormalizeAngle`）。
