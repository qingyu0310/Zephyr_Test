# 电机功率控制算法详解

> **参考开源**：西交利物浦大学 RM2023 电机功率模型 + 香港科技大学ENTERPRIZE RM2024 功率控制  
> **适用场景**：RoboMaster 哨兵机器人底盘功率限制（45W~150W）  
> **核心思想**：通过电机功率模型预测功率 → RLS在线参数整定 → 隶属度分配 → 受限力矩求解

---

## 一、算法总览

功率控制是哨兵底盘中最关键的模块之一。由于 RoboMaster 裁判系统对底盘有严格的功率上限，必须在功率预算内最大化底盘机动性。本项目的功率控制分为**舵向**和**行进**两个独立的子系统（各有4个电机），优先级策略为**舵向优先80%，剩余给行进**。

```
┌─────────────────────────────────────────────────────────────────┐
│                     Power_Ctrl() 功率控制流程                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ① 电机数据采集 ──────────────────────────────────────────────┐  │
│     读取电流Current、转速Velocity(rpm)                           │  │
│     ├─ Current → Torque(转矩) = Current × 4.577e-5            │  │
│     └─ Velocity → Omiga(角速度) = Velocity / 9.55 (rad/s)     │  │
│                                                                │  │
│  ② 功率模型预测 ──────────────────────────────────────────────│  │
│     P_in = K1×τ² + K2×ω² + τ×ω + K3                           │  │
│     (K1,K2,K3 为拟合参数)                                       │  │
│                                                                │  │
│  ③ RLS在线自整定 ─────────────────────────────────────────────│  │
│     用实测功率 vs 预测功率的误差更新 K1,K2                       │  │
│                                                                │  │
│  ④ 功率分配（隶属度） ──────────────────────────────────────│  │
│     综合PID误差占比 + 预测功率占比 → 单电机功率上限               │  │
│                                                                │  │
│  ⑤ 受限力矩求解 ──────────────────────────────────────────────│  │
│     若预测功率 > 分配上限, 解一元二次方程求限制电流值             │  │
│     否则直接输出PID结果                                         │  │
│                                                                │  │
└─────────────────────────────────────────────────────────────────┘
```

## 二、电机功率模型

### 2.1 模型来源

本项目的功率模型参考西交利物浦大学的开源工作。直流电机的功率消耗主要来自三部分：

- **铜损**：与电流平方成正比 → `K1 × τ²`
- **铁损/摩擦损**：与转速平方成正比 → `K2 × ω²`
- **机械功率**：转矩 × 角速度 → `τ × ω`
- **常数项**：静止损耗（电路基础功耗）→ `K3`

### 2.2 模型公式

```
P_in = K1 × τ² + K2 × ω² + τ × ω + K3
```

其中：
| 符号 | 含义 | 计算方式 |
|------|------|----------|
| `τ` | 电机转矩 (N·m) | `τ = Current × 4.577e-5` （3508电机转矩常数） |
| `ω` | 角速度 (rad/s) | `ω = Velocity(rpm) / 9.55` |
| `P_in` | 预测功率 (W) | 模型输出 |
| `K1` | 铜损系数 | RLS在线拟合 |
| `K2` | 铁损系数 | RLS在线拟合 |
| `K3` | 常数偏置 | 经验设定（舵向:3, 行进:3.5） |

### 2.3 代码实现 (Power_Ctrl 数据采集阶段)

```c
for (i = 0; i < 4; i++) {
    // --- 舵向电机 (Motor_Course) ---
    // 电流低通滤波
    LowPassFilter1p_Update(&LPF_Motor[i], Motor_Course[i].Data.Current);
    // 角速度 (rpm → rad/s)
    PowerCtrl_Info[0].Target.Omiga[i] = (Motor_Course[i].Data.Velocity / 9.55);
    // 转矩 = 电流 × 转矩常数
    PowerCtrl_Info[0].Target.Torque[i] = ((LPF_Motor[i].Output) * 4.577e-5);
    // 平方项
    PowerCtrl_Info[0].Target.Omiga_2[i] = powf(PowerCtrl_Info[0].Target.Omiga[i], 2.f);
    PowerCtrl_Info[0].Target.Torque_2[i] = powf(PowerCtrl_Info[0].Target.Torque[i], 2.f);
    // 单电机功率预测
    PowerCtrl_Info[0].Target.Power_In[i] = 
        PowerCtrl_Info[0].K1 * PowerCtrl_Info[0].Target.Torque_2[i] 
        + PowerCtrl_Info[0].K2 * PowerCtrl_Info[0].Target.Omiga_2[i] 
        + PowerCtrl_Info[0].Target.Omiga[i] * PowerCtrl_Info[0].Target.Torque[i];

    // --- 行进电机 (Motor_Advance) --- (同上)
    PowerCtrl_Info[1].Target.Omiga[i] = (Motor_Advance[i].Data.Velocity / 9.55);
    PowerCtrl_Info[1].Target.Torque[i] = ((Motor_Advance[i].Data.Current) * 4.577e-5);
    PowerCtrl_Info[1].Target.Omiga_2[i] = powf(PowerCtrl_Info[1].Target.Omiga[i], 2.f);
    PowerCtrl_Info[1].Target.Torque_2[i] = powf(PowerCtrl_Info[1].Target.Torque[i], 2.f);
    PowerCtrl_Info[1].Target.Power_In[i] = 
        PowerCtrl_Info[1].K1 * PowerCtrl_Info[1].Target.Torque_2[i] 
        + PowerCtrl_Info[1].K2 * PowerCtrl_Info[1].Target.Omiga_2[i] 
        + PowerCtrl_Info[1].Target.Omiga[i] * PowerCtrl_Info[1].Target.Torque[i];
}
```

## 三、RLS 递归最小二乘在线参数整定

### 3.1 原理

RLS (Recursive Least Squares) 是一种在线参数估计算法，核心思想是**用新数据不断更新模型参数**，使预测误差的平方和最小化。

相比批量最小二乘，RLS 的优势在于：
1. **实时性**：每次采样迭代一次，计算量小
2. **自适应**：随着电机老化、温度变化，参数自动调整
3. **遗忘因子 λ**：λ=0.99999，对历史数据指数衰减，适应时变系统

### 3.2 数学模型

对于一个线性系统：
```
y(n) = Wᵀ × X(n) + e(n)
```

RLS更新步骤：

```
① 计算增益 K(n) = P(n-1)×X(n) / [λ + Xᵀ(n)×P(n-1)×X(n)]
② 更新参数 W(n) = W(n-1) + K(n)×[y(n) - u(n)]
③ 更新协方差 P(n) = [P(n-1) - K(n)×Xᵀ(n)×P(n-1)] / λ
```

其中：
- `W = [K1_steer, K2_steer, K1_drive, K2_drive]ᵀ` ← 待辨识参数
- `X = [∑τ₁², ∑ω₁², ∑τ₂², ∑ω₂²]ᵀ` ← 输入特征
- `y = 实测底盘总功率` ← 标签
- `u = 预测功率总和` ← 模型输出
- `λ = 0.99999` ← 遗忘因子
- `P` ← 协方差矩阵，初始化为 `1e-5 × I`

### 3.3 初始化

```c
// X_Size=4, Y_Size=1, Lamda=0.99999, P_Init=1e-5
RLS_Init(&RLS_Power_Info, 4, 1, 0.99999, 1e-5);

// K1_steer, K2_steer, K1_drive, K2_drive 初始值
RLS_Power_Info.Data.W[0] = 1.453e-07;   // 舵向 K1
RLS_Power_Info.Data.W[1] = 1.23e-07;    // 舵向 K2
RLS_Power_Info.Data.W[2] = 1.453e-07;   // 行进 K1
RLS_Power_Info.Data.W[3] = 1.23e-07;    // 行进 K2
```

### 3.4 RLS Update 实现

```c
void RLS_Update(RLS_Info_TypeDef *RLS)
{
    // 1. XT = transpose(X)
    Matrix_Transpose(&RLS->Mat.X, &RLS->Mat.XT);
    
    // 2. Error = Y - U (实测 - 预测)
    Matrix_Subtract(&RLS->Mat.Y, &RLS->Mat.U, &RLS->Mat.E);
    
    // 3. K_Numerator = P * X
    Matrix_Multiply(&RLS->Mat.P, &RLS->Mat.X, &RLS->Mat.K_Numerator);
    
    // 4. Cache = XT * P * X (标量)
    Matrix_Multiply(&RLS->Mat.XT, &RLS->Mat.P, &RLS->Mat.Cache_Matrix[0]);
    Matrix_Multiply(&RLS->Mat.Cache_Matrix[0], &RLS->Mat.X, &RLS->Mat.Cache_Matrix[1]);
    
    // 5. Denominator = 1/(λ + Cache)
    Matrix_Add(&RLS->Mat.Lamda, &RLS->Mat.Cache_Matrix[1], &RLS->Mat.Cache_Matrix[0]);
    Matrix_Inverse(&RLS->Mat.Cache_Matrix[0], &RLS->Mat.K_Denominator);
    
    // 6. K = K_Numerator * Denominator (卡尔曼增益)
    Matrix_Multiply(&RLS->Mat.K_Numerator, &RLS->Mat.K_Denominator, &RLS->Mat.K);
    
    // 7. W_new = W_old + K * Error
    Matrix_Multiply(&RLS->Mat.K, &RLS->Mat.E, &RLS->Mat.Cache_Matrix[0]);
    Matrix_Add(&RLS->Mat.W, &RLS->Mat.Cache_Matrix[0], &RLS->Mat.Output);
    
    // 8. P_new = (P - K*XT*P) / λ
    // ... 协方差矩阵更新
    for (uint8_t i = 0; i < RLS->P_Size; i++)
        RLS->Data.P[i] = (1/RLS->Data.Lamda[0]) * ...;
}
```

> **注意**：当前代码中 `RLS_Update(&RLS_Power_Info)` 被注释掉了，使用手动初始化的 K1,K2 定值。可通过取消注释开启在线自整定。

### 3.5 在 Control_Task 中的数据装配

```c
// 预测功率总和 (作为模型输出 u)
```

## 四、功率分配（隶属度函数）

### 4.1 分配策略

采用港科大ENTERPRIZE战队的方案，核心思路是：**动态调整每个电机的功率上限** ——
既不浪费功率预算，也避免某个电机被过度限制导致失控。

**功率优先级**：
```
舵向(Steer) Power_Max = 0.8 × Chassis_Power_Max    ← 转向优先
行进(Drive) Power_Max = Chassis_Power_Max - |P_steer|  ← 剩余给行进
```

### 4.2 隶属度计算

每个电机的功率分配因子由两部分加权合成：

```
Membership[i] = K × (|Err[i]| / ∑|Err|) + (1-K) × (|P_in[i]| / ∑|P_in|)
```

- **第一部分**：PID误差占比 — 误差大说明该电机需要更多功率
- **第二部分**：当前功率占比 — 当前消耗大说明该电机工作强度高
- **权重 K**：由总误差水平决定

### 4.3 权重 K 的计算

```
K = 0                         当 ∑|Err| < Err_Lower  (误差很小, 按功率分配)
K = 1                         当 ∑|Err| > Err_Upper  (误差很大, 按需求分配)
K = (∑|Err| - Err_Lower)     中间值线性插值
    / (Err_Upper - Err_Lower)
```

### 4.4 代码实现

```c
// 舵向: Err_Upper=50, Err_Lower=0.01
for (int i = 0; i < 4; i++)
    PowerCtrl_Info[0].Err[i] = PID_Chassis_angle_v[i].Err[0];
PowerCtrl_Info[0].Sum.Err_Sum = fabsf(Err[FL]) + fabsf(Err[FB]) + fabsf(Err[RB]) + fabsf(Err[RL]);

if (Sum.Err_Sum > Err_Upper)      K = 1;
else if (Sum.Err_Sum < Err_Lower) K = 0;
else K = (Sum.Err_Sum - Err_Lower) / (Err_Upper - Err_Lower);

// 行进: Err_Upper=500, Err_Lower=0.001
// (同上)

// 计算隶属度并分配功率上限
for (int i = 0; i < 4; i++) {
    Membership[i] = K * (|Err[i]| / ∑|Err|) + (1-K) * (|P_in[i]| / ∑|P_in|);
    VAL_LIMIT(Membership[i], 0, 1);
    Power_Limit[i] = Membership[i] * Power_Max;
}
```

## 五、受限力矩求解

### 5.1 问题描述

当预测总功率超过分配上限时，需要求解**满足功率限制的最大允许力矩值**。

由功率模型：
```
P_limit = K1 × τ² + K2 × ω² + τ × ω + K3
```

转化为标准二次方程：
```
K1 × τ² + ω × τ + (K2 × ω² + K3 - P_limit) = 0
```

其中：
```
A = K1
B = ω
C = K2 × ω² + K3 - P_limit
```

### 5.2 求根公式

```
τ = (-B ± √(B² - 4AC)) / (2A)
```

根据当前输出方向（正/负）选择对应的根，然后通过转矩常数反算电流值。

### 5.3 代码实现

```c
if (Power_Sum >= Power_Max) {
    for (int i = 0; i < 4; i++) {
        A = K1;                             // 铜损系数
        B = Omiga[i];                        // 角速度
        C = Omiga_2[i] * K2 + K3 - Power_Limit[i];  // 常数项
        Delta = B*B - 4*A*C;

        if (isnan(Delta) || isinf(Delta) || Delta < 0)
            Delta = 0;  // 无效时取边界

        if (Delta >= 0) {
            Sqrt = sqrt(Delta);
            if (当前输出 >= 0)
                Torque = (-B + Sqrt) / (2*A);
            else
                Torque = (-B - Sqrt) / (2*A);  // 负方向取负根
            
            输出电流 = Torque / 4.577e-5;  // 转矩→电流转换
        }
    }
}
else {
    // 功率未超限，直接使用PID输出
    输出电流 = PID_Output;
}
```

### 5.4 输出反转逻辑

- **舵向电机**（3508角度轮）：PID输出为角度环→速度环串级输出，功率限制后替换角度值
- **行进电机**（3508行进轮）：PID输出为速度环输出，功率限制后替换速度值

关键转换关系：
```
电流值 = 转矩 / 4.577e-5
```
其中 `4.577e-5` 是 M3508 电机的转矩常数（N·m/A）。之所以能直接用这个值，是因为 M3508 的电流控制模式是通过 `current × torque_constant` 映射到电磁转矩，这是 RoboMaster 电机驱动器的标准接口。

## 六、超级电容功率管理

### 6.1 原理

超级电容提供额外的功率缓冲，使哨兵能够短时间超过裁判系统的功率上限。

### 6.2 控制逻辑

```c
void Control_SuperCap_Mode_Update(Control_Info_Typedef *Control_Info) {
    if (Chassis_Info.Supercap.Persent >= 15) {
        // 电容电量充足: 功率上限 = 裁判系统限制 + 100W 缓冲
        Chassis_Info.Supercap.Power_Max = 
            Referee_System_Info.robot_status.chassis_power_limit 
            - PID_Buffer.Output + 100;
    }
    else {
        // 电容电量不足: 仅使用裁判系统限制
        Chassis_Info.Supercap.Power_Max = 
            Referee_System_Info.robot_status.chassis_power_limit 
            - PID_Buffer.Output;
    }
    Chassis_Info.Power_Max = Chassis_Info.Supercap.Power_Max;
}
```

其中 `PID_Buffer` 是一个独立的PID控制器，用于处理裁判系统的buffer能量值，使得功率消耗更加平滑：

```c
// buffer能量的PID控制: 目标60J, 反馈当前buffer值
PID_Calculate(&PID_Buffer, 60, Referee_System_Info.power_heat_data.buffer_energy);
PID_Buffer_Param = {3, 0, 0, 0, 0, 0, 60};  // P=3, Output_Max=60
```

## 七、低通滤波

### 7.1 一阶低通滤波器

用于平滑电流和功率信号，滤除高频噪声。

```
输出 = Alpha × 上次输出 + (1 - Alpha) × 本次输入
```

| 滤波器 | Alpha | 用途 |
|--------|-------|------|
| `LPF_ChassisPower` | 0.7 | 实测底盘总功率平滑 |
| `LPF_Motor[i]` | 0.1 | 各电机电流平滑 |

Alpha 越小滤波越平滑但延迟越大，Alpha 越大响应越快但噪声抑制弱。

### 7.2 PID微分项滤波

PID的D项输出经过一阶低通滤波，减少微分放大噪声的影响：

```c
// 在PID计算中
PID->Dout = LowPassFilter1p_Update(&PID->Dout_LPF, PID->Dout);
```

## 九、算法流程总结

```
每个控制周期(1ms):
  │
  ├─ 读取电机电流、转速
  ├─ 低通滤波电流值
  ├─ 计算单电机预测功率 P_in = K1τ² + K2ω² + τω + K3
  ├─ 求和得总预测功率
  │
  ├─ [RLS在线更新]
  │   ├─ 装配数据: X=[∑τ₁²,∑ω₁²,∑τ₂²,∑ω₂²], y=P_实测
  │   ├─ RLS_Update() 更新K1,K2
  │   └─ 提取新参数 (当前被注释)
  │
  ├─ 计算隶属度并分配功率上限
  │   ├─ 读取PID误差
  │   ├─ 计算权重K (插值)
  │   └─ Membership[i] → Power_Limit[i]
  │
  ├─ 判断是否超功率
  │   ├─ 是 → 解二次方程求限制力矩并换算为电流
  │   └─ 否 → 直接使用PID输出值
  │
  └─ 输出最终电流值到CAN总线
```
