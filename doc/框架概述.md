# MotorCtrl 框架概述

基于 Zephyr RTOS 的电机控制框架，目标架构为解耦的三层模型：**中断响应** → **算法控制** → **总线发送**，层间通过 Zephyr **zbus** 发布订阅通信。

---

## 一、架构总览

```
┌──────────────────────────────────────────────────────────┐
│                        zbus                              │
│  ┌─────────────────────┐    ┌──────────────────────┐     │
│  │     MotorFB         │    │     MotorCMD          │     │
│  │  (反馈数据通道)      │    │  (控制指令通道)       │     │
│  └───────┬─────────────┘    └─────────┬────────────┘     │
│          │                            │                   │
│  ┌───────┴─────────────┐    ┌─────────┴────────────┐     │
│  │   C6xx RX (中断)    │    │  Chassis (1ms 线程)  │     │
│  │   CAN 回调 → 解析   │    │  PID + PowerCtrl     │     │
│  │   zbus_pub(MotorFB) │    │  zbus_pub(MotorCMD)  │     │
│  └─────────────────────┘    └─────────┬────────────┘     │
│                                       │                   │
│                               ┌───────┴────────────┐     │
│                               │   CAN_TX (线程)     │     │
│                               │  zbus_sub(MotorCMD) │     │
│                               │  can_send()         │     │
│                               └────────────────────┘     │
└──────────────────────────────────────────────────────────┘
```

### 三层职责

| 层 | 上下文 | 周期 | 职责 |
|----|--------|------|------|
| **中断层** | CAN 中断 | 硬件触发 | 解析 CAN 帧 → 写 seqlock 保护的数据 |
| **控制层** | Chassis 线程 | 1~2ms | PID + 功率限制 → 输出控制指令 |
| **发送层** | CAN_TX 线程 | 阻塞等待 | 订阅指令 → 组帧 → 发送 |

## 二、数据流

### 接收路径（中断上下文）

```
CAN RX 中断
  └── mcan4_rx_callback_func(frame)
        └── C6xx.CanCpltRxCallback(rx_data)
              ├── 解析: enc/omega_rpm/current_raw/temp
              ├── 计算: angle/omega/current/torque/velocity
              └── seqlock 保护写入成员变量
```

C6xx 内部使用 **seqlock**（`atomic_t seq_`）保证控制线程读到一致快照。控制线程调用 `ReadAll()` 时自旋等待直到读到前后一致的 seq。

### 控制循环（Chassis 线程）

```
每个控制周期 (2ms):
  ├── motors1.ReadAll()           ← 原子读取电机快照
  ├── pidCtrl.Calc(target, now)   ← PID 计算目标电流
  ├── pwrCtrl.SetTarget(idx, raw) ← 串级：PID 输出给功率控制器
  ├── pwrCtrl.SetMotorData(...)   ← 喂实测数据
  ├── pwrCtrl.Predict()           ← 功率模型预测
  ├── pwrCtrl.Allocate(budget)    ← 功率分配 + 限制
  ├── msg.SetOut(idx, limited)    ← 取限制后的电流值
  └── zbus_chan_pub(MotorCMD)     ← 发布到 zbus
```

### 发送路径（CAN_TX 线程）

```
CAN_TX 线程:
  └── zbus_sub_wait(can_tx_sub)  ← 阻塞等待 MotorCMD
        └── zbus_chan_read(msg)  ← 读取最新指令
              └── 组 can_frame → mcan4.Send()
```

## 三、线程模型

| 线程 | 优先级 | 周期 | 文件 |
|------|--------|------|------|
| Chassis | 5 | 2ms | `projects/thread/chassis/trd_chassis.cpp` |
| CAN_TX | 5 | 阻塞 | `projects/thread/can/trd_can_tx.cpp` |
| GPIO | 5 | 按需 | `projects/thread/gpio/trd_gpio.cpp` |

所有线程使用统一的 `Thread<>` 模板（`projects/thread/thread.hpp`），封装 `k_thread_create` 样板代码。

### 线程注册规范

每个线程提供两个函数，在 `System_startup.cpp` 中调用：

```cpp
namespace thread::xxx {
    inline Thread<> thread_{};
    void thread_init();          // 初始化：配置驱动、PID 参数等
    void thread_start(uint8_t prio, ...);  // 启动线程
};
```

## 四、发布订阅（zbus）

### MotorCMD 通道

定义在 `topic/motor_cmd.hpp`：

```cpp
struct motor_cmd_msg {
    uint16_t can_id;         // CAN 控制器索引
    uint16_t tx_id;          // CAN 帧 ID
    uint8_t data[8];         // 8 字节数据

    void SetOut(uint8_t idx, float current_A);  // 设置电流 (A → 原始值)
    float GetOut(uint8_t idx) const;             // 读取电流 (原始值 → A)
};
```

- 发布：Chassis 线程
- 订阅：CAN_TX 线程

### Kconfig 控制

`topic/Kconfig` 中定义 `TPC_MOTOR_CMD`，被 `TRD_CHASSIS` 和 `TRD_CAN_TX` 自动 `select`。

## 七、算法管线

### PID 控制器 (`algorithm/controller/pid/`)

位置式 PID，头文件即文档。

**功能**：
- 微分先行（D on measurement，避免 setpoint kick）
- 变速积分（误差大时减弱积分强度）
- 积分分离（大误差时清零积分防饱和）
- D 项低通滤波
- 角度模式（-π~π 劣弧自动处理）
- 双缓冲参数更新（`SetShadow`，运行时安全修改）

**使用**：

```cpp
alg::pid::Pid pid({.kp = 3.0f, .ki = 0.1f, .kd = 0.05f, .outMax = 60.0f});
float out = pid.Calc(target, feedback);
```

运行时调参：`pid set kp 5.0`（shell），或 `pidCtrl.SetShadow(cfg)`（代码）。

### 功率控制器 (`algorithm/controller/power_ctrl/`)

直流电机功率模型，限制底盘总功率不超过预算。

**模型**：
```
P_in = K1·τ² + K2·ω² + τ·ω(可选) + K3
```

**管线**：
1. `SetMotorData(idx, torque, omega, err)` — 输入实测值
2. `Predict()` — 功率预测 + RLS 辨识（可选）
3. `Allocate(budget)` — 隶属度分配 + 力矩限制
4. `GetLimitedCurrent(idx)` — 获取限制后的电流

**功能**：
- 单组或多组电机管理（`motorCount`）
- RLS 在线参数辨识（`rlsEnable`）
- `τ·ω` 项开关（`tauOmegaEnable`，转向电机建议关闭）
- 隶属度分配（综合 PID 误差 + 当前功率动态分配预算）

**使用**：

```cpp
alg::power_ctrl::PowerCtrl ctrl({
    .motorCount = 4,
    .k3 = 3.0f,
    .tauOmegaEnable = false,  // 转向电机建议关
});
ctrl.SetMotorData(i, snap.torque, snap.omega, pidCtrl.GetError());
ctrl.Predict();
ctrl.Allocate(60.0f);
float out = ctrl.GetLimitedCurrent(i);
```

### RLS 辨识 (`algorithm/identify/rls/`)

递归最小二乘，2 输入 1 输出，遗忘因子 λ=0.99999。

在 PowerCtrl 内部自动管理，通常在电机运动数据足够时开启（`rlsEnable=true`），收敛后记下 K1/K2 作为 init 值，关掉 RLS 使用定值。

## 九、如何添加新线程

1. 在 `projects/thread/` 下创建目录，如 `newtask/`
2. 实现 `trd_newtask.hpp`：声明 `thread::newtask` 命名空间 + 线程实例
3. 实现 `trd_newtask.cpp`：`Task()` + `thread_init()` + `thread_start()`
4. 在 `projects/thread/Kconfig` 添加 `TRD_NEWTASK`，`select` 所需组件
5. 在 `projects/thread/CMakeLists.txt` 添加条件编译
6. 在 `projects/apps/System_startup.cpp` 调用 `thread_init()` / `thread_start()`
