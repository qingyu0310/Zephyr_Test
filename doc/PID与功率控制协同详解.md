# PID 与功率控制协同详解

> **核心观点**：PID 和功率控制不是并列关系，而是 **PID 负责"该给多少"，功率控制负责"能给的限制是多少"**。功率控制是一个**约束层**，在 PID 输出之后做检查，超限就替换，不超限就透传。

---

## 一、整体数据流

```
                    ┌──────────────────┐
                    │  遥控器 / MiniPC  │
                    │  Vx, Vy, Vw      │
                    └────────┬─────────┘
                             │
                             ▼
                    ┌──────────────────┐
                    │ 舵轮运动学解算    │
                    │ → 4个目标角度     │
                    │ → 4个目标速度     │
                    └────────┬─────────┘
                             │
                             ▼
                    ┌──────────────────┐
                    │   PID 控制器     │ ◄── ① PID 先算
                    │                   │
                    │ 舵向角度环(P=62)  │
                    │ 舵向速度环(P=20)  │
                    │ 行进速度环(P=15)  │
                    │ 云台Follow PID    │
                    │ Buffer PID(P=3)   │
                    └────────┬─────────┘
                             │ PID_Output (原始期望电流)
                             ▼
                    ┌──────────────────┐
                    │  功率控制器      │ ◄── ② 功率再限制
                    │  Power_Ctrl()    │
                    │                   │
                    │ ① 读电流/转速     │
                    │ ② 预测总功率      │
                    │ ③ 隶属度分配      │
                    │ ④ 超限？          │
                    │    ├─ 是 → 解方程 │
                    │    └─ 否 → 透传   │
                    └────────┬─────────┘
                             │ Final_Output (最终电流)
                             ▼
                    ┌──────────────────┐
                    │  CAN 总线 → 电机  │
                    └──────────────────┘
```

**一句话总结**：PID 输出是"需求"，功率控制是"审批"。预算够就批，不够就砍到预算内。

## 二、PID 在哪几个位置参与协同？

本系统中 PID 与功率控制有 **4 个协同点**：

### 协同点 1：PID 误差 → 隶属度分配（功率分配的依据）

功率控制里的**隶属度函数**直接读取 PID 的实时误差来决定"谁更需要功率"：

```c
// Power_Ctrl() 中读取 PID 误差
for (int i = 0; i < 4; i++) {
    // 舵向：读取角度速度环的误差
    PowerCtrl_Info[0].Err[i] = PID_Chassis_angle_v[i].Err[0];
    // 行进：读取速度环的误差
    PowerCtrl_Info[1].Err[i] = PID_Chassis_velocity[i].Err[0];
}

// 隶属度 = 误差占比 × K + 功率占比 × (1-K)
Membership[i] = K * (|Err[i]| / ∑|Err|) + (1-K) * (|P_in[i]| / ∑|P_in|);
```

**协同逻辑**：
- **PID 误差大** → 这个电机急需纠正 → 隶属度高 → 分到更多功率预算
- **PID 误差小** → 这个电机已经到位 → 隶属度低 → 少分功率，留给其他电机

这本质上是 **PID 误差驱动资源分配**：功率预算不是平均分，而是按需分配，"需"就是 PID 误差。

### 协同点 2：功率限制 → 替换 PID 输出（硬约束）

这是最直接的协同。功率控制运行在 PID 计算之后：

```c
// PID 先算 —— Control() 函数
Control(&Chassis_Info);
// → PID_Chassis_angle_v[i].Output  (舵向原始电流值)
// → PID_Chassis_velocity[i].Output  (行进原始电流值)

// PID 结果赋给输出 —— Control_Info_Update()
Chassis_Info.Output.Chassis_Angel[i] = PID_Chassis_angle_v[i].Output;
Chassis_Info.Output.Chassis_Velocity[i] = PID_Chassis_velocity[i].Output;

// 功率控制后执行 —— Power_Ctrl()
Power_Ctrl();
// → 如果超功率，直接覆盖 Output 数组
// → 如果不超功率，Output 保持 PID 原值不变
```

**超限时的替换逻辑**：

```
PID算完 → Output = PID值       ← 先放PID的结果
         ↓
Power_Ctrl检查总预测功率是否超限
         ↓
    超限？──→ 是 ──→ 解二次方程求限制力矩
         │           τ = (-ω + √Δ) / (2×K1)
         │           Output = τ / 4.577e-5    ← 覆盖为限制值
         │
         └── 否 ──→ Output 保持 PID 原值     ← 透传

最终发送到 CAN 总线的就是 Output 数组
```

**关键理解**：PID 输出只是"建议值"，功率控制有最终决定权。

### 协同点 3：PID_Buffer → 功率预算调节（外环修正）

超级电容管理中，**用一个独立的 PID 控制器来调节总功率预算**：

```c
// 这是一个外环 PID
// 目标值 = 60J (buffer能量目标)
// 反馈值 = 裁判系统当前 buffer 能量
PID_Calculate(&PID_Buffer, 60, Referee_System_Info.power_heat_data.buffer_energy);
// PID_Buffer 参数: P=3, I=0, D=0, Output_Max=60

// 用 PID 的输出修正功率上限
Chassis_Info.Supercap.Power_Max = 
    Referee_System_Info.robot_status.chassis_power_limit 
    - PID_Buffer.Output     // ← PID修正项
    + 100;                  // ← 超电缓冲
```

**协同逻辑**：

| buffer 能量状态 | PID_Buffer.Output | 效果 |
|----------------|-------------------|------|
| buffer > 60J（过剩） | 负值 | 功率上限提升，多用电 |
| buffer < 60J（不足） | 正值 | 功率上限降低，省着用 |
| buffer = 60J（平衡） | 0 | 功率上限 = 裁判限制 + 100W |

这是一个 **能量管理外环 PID**，它控制的是"功率预算"本身，而不是直接控制电机。

### 协同点 4：PID 微分项滤波（信号质量）

PID 控制器中的微分项经过低通滤波，减少噪声放大：

```c
// pid.c 中的 D 项计算
PID->Dout = LowPassFilter1p_Update(&PID->Dout_LPF, PID->Dout);
```

这间接影响功率控制：**更平滑的 PID 输出 → 更稳定的电流 → 更准确的功率预测 → 更合理的功率限制**。

## 三、时序关系（1ms 控制周期内）

```
时间轴 (1ms)
│
├── [t=0]   读取电机编码器/IMU (Control_Measure_Update)
│
├── [t=0.1] 运动学解算目标值 (Control_Target_Update)
│
├── [t=0.2] PID 计算 (Control)
│   ├── 读取 8 个电机的误差 Err[0]
│   ├── 舵向角度环 PID → 舵向速度环 PID
│   ├── 行进速度环 PID
│   └── 输出 PID_Output (原始期望电流)
│
├── [t=0.3] PID 输出赋给 Output 数组 (Control_Info_Update)
│
├── [t=0.4] 超级电容模式更新 (Control_SuperCap_Mode_Update)
│   └── PID_Buffer 计算 → 确定 Power_Max
│
├── [t=0.5] 功率控制 (Power_Ctrl)
│   ├── 读取当前电流/转速（用于功率预测）
│   ├── 读取 PID_Chassis_angle_v[i].Err[0]  ← 协同点①
│   ├── 读取 PID_Chassis_velocity[i].Err[0]  ← 协同点①
│   ├── 计算隶属度 → 功率上限
│   ├── 判断是否超限
│   │   ├── 超限 → 解方程替换 Output       ← 协同点②
│   │   └── 不超限 → 保持 PID_Output 不变   ← 协同点②
│   └── 写回 Chassis_Info.Output[]
│
└── [t=0.6] CAN 发送 Output[] 到电机 (CAN_Task)
```

**一个周期内，PID 和功率控制的角色转换**：

```
控制周期开始          控制周期结束
     │                    │
     ▼                    ▼
  PID 是主角         功率控制是主角
  (计算期望)         (判断能否执行)
     │                    │
     └────── 交接点 ──────┘
     在 Control_Info_Update 之后,
     Output 数组从 PID 手中交给 Power_Ctrl
```

## 四、一个具体的协同例子

### 场景

底盘功率限制 **45W**，舵向组 `Power_Max = 36W`（80%）。哨兵正在快速转弯，4 个舵向电机状态如下：

| 电机 | 当前电流 | 转速 | 预测功率 | PID 误差 |
|------|---------|------|---------|---------|
| FL | 3A | 2000rpm | 12W | **30** ← 误差很大（被撞偏了）|
| FR | 1A | 500rpm | 5W | 5 |
| RL | 2A | 1000rpm | 8W | 8 |
| RR | 1A | 800rpm | 6W | 4 |

**步骤 1：PID 先算**

FL 的舵向角度环看到误差 30°，输出一个大电流想把轮子掰回来。

```c
PID_Chassis_angle_v[FL].Output = 7200;  // 期望电流值
```

**步骤 2：功率控制检查**

总预测功率 = 12 + 5 + 8 + 6 = **31W** ≤ 36W（上限）。没超，透传：

```
Output[FL] = 7200  ← 保持 PID 原值
```

### 换个场景

逼近功率上限时，总预测功率 = 12 + 8 + 10 + 9 = **39W > 36W**。

**步骤 1：隶属度分配（K 值计算）**

∑|Err| = 30 + 5 + 8 + 4 = 47，舵向 Err_Upper=50, Err_Lower=0.01：

```
K = (47 - 0.01) / (50 - 0.01) = 0.94  ← 误差大，主要按需分配
```

**步骤 2：单电机功率上限**

```
Membership[FL] = 0.94 × (30/47) + 0.06 × (12/31) = 0.623
Power_Limit[FL] = 0.623 × 36W = 22.4W
```

FL 分到 22.4W（因为它误差最大）。

**步骤 3：解方程求限制力矩**

```
A = K1 = 1.453e-07
B = ω = 2000/9.55 = 209.4 rad/s
C = K2×ω² + K3 - Power_Limit = 1.23e-07×43838 + 3 - 22.4 = -13.9

Δ = 209.4² - 4×1.453e-07×(-13.9) = 43838 + 8.08e-06 ≈ 43838

τ = (-209.4 + √43838) / (2×1.453e-07) = (-209.4 + 209.4) / 2.906e-07 ≈ 0.04 N·m
```

**步骤 4：替换 PID 输出**

```
Output[FL] = 0.04 / 4.577e-5 = 874  ← 从 7200 被砍到 874
```

**对比**：

| 电机 | PID 期望电流 | 功率限制后电流 | 说明 |
|------|------------|--------------|------|
| FL | 7200 | **874** | 误差最大但也被限得最狠 |
| FR | 2800 | 2800 | 没超限，透传 |
| RL | 3500 | 3100 | 轻微限制 |
| RR | 3000 | 3000 | 没超限，透传 |

**FL 虽然分到的功率预算最多（22.4W），但仍被大幅限流**。这就是功率控制的核心矛盾：**PID 说"我需要更多"，功率控制说"你只能拿这么多"**。

## 五、协同关系总结

```
                    PID 控制器
                  ┌────────────┐
                  │ 角度环     │
                  │ 速度环     │
                  └──────┬─────┘
                         │ PID 误差 Err[i]
            ┌────────────┼──────────────┐
            │            │              │
            ▼            ▼              ▼
     ┌──────────┐  ┌──────────┐  ┌──────────┐
     │ 隶属度   │  │ PID_Buffer│  │ 最终输出  │
     │ 分配     │  │ (外环PID) │  │ 裁决     │
     │          │  │          │  │          │
     │ Err占比  │  │ 目标60J  │  │ 超限？   │
     │ → 功率   │  │ → 修正   │  │ ├→ 替换  │
     │   上限   │  │   功率   │  │ └→ 透传  │
     └──────────┘  └──────────┘  └──────────┘
          │              │              │
          └──────┬───────┘              │
                 │                      │
                 ▼                      ▼
          功率预算决策             最终电流值
          (谁拿多少)               (发到CAN总线)
```

| 协同点 | PID 角色 | 功率控制角色 | 数据接口 |
|--------|---------|-------------|---------|
| ① 隶属度分配 | 提供误差 Err[i] | 按误差占比分配功率 | `PID_xxx.Err[0]` |
| ② 输出替换 | 提供期望电流 Output | 超限时解方程替换 | `Chassis_Info.Output[]` |
| ③ Buffer 管理 | PID_Buffer 控制充放 | 修正总功率预算 | `PID_Buffer.Output` |
| ④ 信号质量 | D 项低通滤波 | 获得更稳定的功率信号 | `PID->Dout_LPF` |

**最终结论**：PID 是 **需求侧**，功率控制是 **供给侧**。功率控制不关心 PID 怎么算的，只关心两件事：
1. **总量够不够**（预测总功率 > 预算？）
2. **谁最需要**（隶属度分配）

总量不够 → 砍到预算内，但优先保"误差大"的电机。
总量够用 → 完全不干预，PID 说了算。

这就是 PID 和功率控制的协同本质。
