# 底盘控制管线（Chassis Control Pipeline）

## 概述

底盘控制是一个 **单线程、2ms 周期、五阶段流水线**。每周期顺序执行：

```
遥控器输入 → 运动学解算 → 串级 PID → 功率分配 → CAN 发布
```

代码位于 `projects/thread/chassis/trd_chassis.cpp`，配置和数据定义在 `trd_chassis.hpp`。

---

## 一、控制周期

```cpp
static constexpr uint32_t kPeriodMs = 2;   // 控制周期 ms
```

循环末尾用 `k_uptime_get()` 测量实际耗时，动态补偿 sleep，防止周期漂移：

```cpp
const int64_t elapsed = k_uptime_get() - tick_start;
const int64_t remain = static_cast<int64_t>(kPeriodMs) - elapsed;
if (remain > 0) {
    k_msleep(remain);
}
```

若某次循环超时（`remain <= 0`），跳过 sleep 直接进入下一帧。

---

## 二、五阶段流水线

### 阶段 1：`ReadRemote()` — 读取遥控器

非阻塞订阅 zbus 通道 `pub_dr16_to`：

```cpp
zbus_sub_wait(&sub_dr16_to, &chan, K_NO_WAIT);
if (chan) {
    zbus_chan_read(chan, &msg_remote_to_chassis, K_NO_WAIT);
    g_vx = msg_remote_to_chassis.axis.x * KMaxMoveVelocity;
    g_vy = msg_remote_to_chassis.axis.y * KMaxMoveVelocity;
    g_vw = msg_remote_to_chassis.axis.yaw * KMaxRotationOmega;
}
```

- 遥控器更新频率约 70Hz（14ms），控制频率 500Hz（2ms）
- 无新数据时保持上一帧的 `g_vx/g_vy/g_vw`（sample-and-hold）
- **输入**：zbus 通道 `pub_dr16_to`
- **输出**：`g_vx, g_vy, g_vw`（底盘速度指令）

### 阶段 2：`UpdateTarget()` — 运动学解算 + 优劣弧

**2a. `SwerveKinCalculate()` — 刚体运动学逆解**

每个轮子的速度 = 底盘平动速度 + 旋转切向速度：

```cpp
vx_w = g_vx - g_vw * y_w    // ω × r 在 x 方向的分量
vy_w = g_vy + g_vw * x_w    // ω × r 在 y 方向的分量
```

轮位配置（匿名 struct 数组）：

```cpp
struct { float x, y; } static constexpr kWheelPos[N_Wheel] {
    {0.0f,  0.135f},   // 轮0: 前 (y = +R)
    {0.0f, -0.135f},   // 轮1: 后 (y = -R)
};
```

目标角度和速度：

```cpp
g_wh_target[wi].angle    = kAngleSign[wi] * atan2f(vx_w, vy_w);
g_wh_target[wi].velocity = hypot(vx_w, vy_w) * kSwerveSign[wi];
```

**2b. `OptimalArc()` — 优劣弧判断**

读取转向电机当前角度，判断目标角度路径：

- 角度差 ≤ 90°：走优弧，行进电机正转
- 角度差 > 90°：目标翻转 ±π，行进电机反转（走短路径）

返回值 `k_factor`（±1）在阶段 3 中乘以行进速度目标。

### 阶段 3：`ControlCalculate()` — 串级 PID

每轮两个串级控制通道：

```
转向链：角度环(P=3.0) → 力矩环(P=0.5) → /torqueK → 目标电流
行进链：速度环(P=1.0) → 力矩环(P=0.3) → /torqueK → 目标电流
```

- 外环输出作为内环目标值（`torque_ref`）
- 内环输出除以 `kTorqueK`（0.3）得到目标电流
- 目标电流送入 `PowerCtrl.SetTarget()`

### 阶段 4：`PowerAlloc()` — 功率分配

使用 RLS 在线辨识的功率模型，分两组独立控制：

| 分组 | 分配策略 | 模型参数 |
|------|----------|----------|
| 转向组 `SteerPwrCtrl` | 分配 80% 预算 | k1=1.45e-7, k2=5.17e-3, k3=3.0 |
| 行进组 `DrivePwrCtrl` | 分剩余 20% | k1=1.45e-7, k2=5.17e-3, k3=3.5 |

流程：

```cpp
turnPred = SteerPwrCtrl.Predict();       // 功率预测
SteerPwrCtrl.Allocate(turnMax);          // 转向最多用 80%

runBudget = kTotalBudget - turnUsed;     // 行进分剩下的
DrivePwrCtrl.Allocate(runBudget);
```

- 有功率计（`USE_POWERMETER=1`）时：输入实测功率给 RLS 做在线辨识
- 无功率计时：使用固定 K1/K2 做开环预测

### 阶段 5：`FramePublish()` — CAN 帧发布

取出功率限制后的电流，按 CAN ID 索引填入数据帧，发布到 zbus：

```cpp
msg_chassis_to_can.SetOut(kSteerDataIdx[wi], SteerPwrCtrl.GetLimitedCurrent(wi));
msg_chassis_to_can.SetOut(kDriveDataIdx[wi], DrivePwrCtrl.GetLimitedCurrent(wi));
zbus_chan_pub(&pub_chassis_to_can, &msg_chassis_to_can, K_MSEC(1));
```

CAN TX 线程（`trd_can_tx.cpp`）订阅该通道，收到后组帧发送到物理 CAN 总线。

## 三、初始化流程

在 `thread_init()` 中完成：

1. **功率计初始化**（可选）— `SteerPwrMeter.Init()`, `DrivePwrMeter.Init()`
2. **功率控制器初始化** — 配置 K1/K2、RLS 开关、隶属度上下限
3. **电机 + PID 初始化** — 每轮先转向后行进，配置 CAN ID、减速比、PID 参数

## 五、扩展指南

### 5.1 增加轮数（2 轮 → 4 轮）

**`trd_chassis.hpp`** 修改三处：

```cpp
// ① 轮数
constexpr uint8_t N_Wheel = 4;

// ② CAN ID（按实际接线补充）
constexpr uint16_t kSteerCanId[N_Wheel]   = {0x202, 0x204, 0x206, 0x208};
constexpr uint16_t kDriveCanId[N_Wheel]   = {0x201, 0x203, 0x205, 0x207};
```

> 注意：`kDriveDataIdx` / `kSteerDataIdx` 由 CAN ID 自动计算，无需手动修改。

**`trd_chassis.cpp`** 修改：

```cpp
// ③ 轮子坐标（加两个轮子）
struct { float x, y; } static constexpr kWheelPos[N_Wheel] {
    { 0.135f,  0.135f},   // 轮0: 前左
    {-0.135f,  0.135f},   // 轮1: 前右
    { 0.135f, -0.135f},   // 轮2: 后左
    {-0.135f, -0.135f},   // 轮3: 后右
};
```

> 运动学公式 `V_wheel = V_chassis + ω × r_wheel` 是 N 轮通用的，不限制轮数。

### 5.2 换用不同的电机

- 修改 `kTorqueK`（转矩常数）
- 修改 `DjiC6xx::Config` 中的 `gearbox_ratio`（减速比）和 `wheel_r`（轮半径）
- 更换电机类（非 DJI）则需要实现相同的 `ReadAll()` 接口（返回 `{angle, omega, torque, velocity, ...}`）

### 5.3 更改控制周期

只需改一处：

```cpp
static constexpr uint32_t kPeriodMs = 2;   // → 改为 1 就是 1ms 周期
```

### 5.4 调 PID

修改 `trd_chassis.cpp` `thread_init()` 中对应的 PID `Config`：

```cpp
alg::pid::Pid::Config angle_pid_cfg{};
angle_pid_cfg.kp  = 3.0f;
angle_pid_cfg.ki  = 0.1f;    // 加 I 项消除静差
angle_pid_cfg.kd  = 0.0f;
```

也可通过 shell 运行时调参（`pid set ...`）。

### 5.5 启用/禁用功率计

`trd_chassis.hpp` 中：

```cpp
#define USE_POWERMETER      0   // 禁用
#define USE_POWERMETER      1   // 启用
```

禁用时功率控制器使用固定 K1/K2 做开环预测，不依赖硬件功率计。

### 5.6 电机方向校准（新底盘）

修改 `kAngleSign[]` 和 `kSwerveSign[]`，参考 `doc/底盘运动学解算说明.md`。

### 5.7 功率模型参数初始化

初次使用功率控制时，在 RLS 开启状态下运行底盘，让 RLS 收敛。收敛后将打印出的 K1/K2 写入 `thread_init()` 的 `kSteerk1Init` / `kSteerk2Init`，然后关闭 RLS（`rlsEnable = false`）使用定值。
